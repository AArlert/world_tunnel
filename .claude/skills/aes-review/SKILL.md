# 审美验收流程（aes 验收职专用；orch 派卡时在卡内引用本 skill）

判据「高级感足够」的可操作化流程。验收对象 = 视觉类交付（globe 风格/标记/面板/动效）。

## 1. 截图取证（先取证后评审，无截图不评审）

- 起 dev server（`make dev` 或 Playwright 对 `http://localhost:5173`），用 Playwright 脚本按**标准机位矩阵**截图：
  - 相机距离三档：远（整球）/ 中（大洲）/ 近（国家级）——对应 D24 LOD 三层；
  - 昼夜两侧：昼半球中心视角 / 夜半球中心视角 / 晨昏线跨越视角；
  - 数据两态：真实当日事件 / 空态（如可构造）；
  - 视口两档：桌面 1440×900 / 移动 390×844（M5 前移动档可缺省，缺省须在记录注明）。
- 截图存 `doc/attachment/aes-<YYYYMMDD>-<主题>/`，文件名含机位（如 `far-day.png`）。一次性脚本放 scratchpad，不入库。

## 2. 评审维度（逐维度给结论，禁止只给总评）

层次/对比 · 配色克制 · 注记与标记密度 · 动效克制 · 面板质感 · 无障碍对比度（正文级文本对底 ≥ 4.5:1，注记级 ≥ 3:1）· 角色卡「3D 地球专属判据」全部条目。

## 3. 分级与门禁

- **Blocker（硬否决）**：命中反模式黑名单 1–5 条之一；半球无可辨明暗对比；任一机位 3 秒读不出 D23 L0 三变量；语义色（SPEC-3.7）被改动色相。
- **High**：层次存在但主次颠倒；密度超载但未糊死；无障碍对比不达标。
- **Medium/Nit**：质感细节、过渡宽度、字距等打磨项。
- 门禁只挡 Blocker/High；Medium/Nit 放行并列入记录供后续批次。

## 4. 记录与闭环

- 记录写 `doc/review/AES-<NNN>-<主题>.md`（NNN 全局递增，与 REV 序列独立）：先「做对了什么」，再逐条问题（分级 + 截图引用 + 判据出处），只提问题不开药方。
- 拦截后的复评 = orch 另派新 aes 实例，凭同一机位矩阵重截对比；复评记录引用前次 AES 编号。
- 视觉场景的完整门禁次序：**aes 验收 → rev 审查/门禁**；aes 管好不好看，rev 管合不合 spec，互不代职。
