## 一、定位(D1–D5)

- **D1 产品定位**:glanceable「世界的表盘」——为所有想看见世界正在发生什么的人,在碎片瞬间与常显场景,打开三秒看见此刻地球上正在发生什么(且只看自己关心的部分)。不替代信息流,给信息流装一张地图。核心价值 = 感知格局(掌控感、全局观),空间关系本身携带信息。
- **D2 一个产品,N 张面孔**:产品机制通用(自选信源/分类/图层),对外营销按垂类分头打(冲突观察 / 航天 / 灾害等)。目标用户不限于爱好者圈层。
- **D3 界面基调**:宁静。警报感/星球感由用户所选内容决定,App 忠实呈现,但界面本身必须令人平静。球为本体、列表从属(维持 SPEC-2.2 主从关系)。
- **D4 命名**:产品名 **Worlens**(上架前完成商标与 App Store 重名检查——待办);中文 slogan 定案**「看见世界正在发生」**;英文 slogan 待办。仓库内部代号保留 world_tunnel,对外(含公开源码仓库)用 Worlens。
  **修订(2026-07-20,依调研 doc/research/market-202607.md,用户拍板)**:发现在运营产品 WorldLens Live 与 Worlens 近同名且近同构,重名检查由「上架前待办」**前移为近期高优任务**;同步启动备选名征集;名称定稿前对外物料(公开仓库名、slogan 铺开)冻结。
  **二次修订(2026-07-20,命名核查 doc/research/naming-202607.md 呈报后,用户拍板)**:**保留 Worlens**——知情接受与 WorldLens Live 的混淆风险,定位贴合优先,备选名(Worldial/Orbrief/Panorb)留档不启用。名称定稿,对外物料冻结解除,但铺开前须完成:①正式商标检索(美/中+目标市场)与域名 WHOIS 核验;②WorldLens Live 混淆度评估与差异化策略。正式检索若见阻断性冲突,触发本条三次修订。
- **D5 用户验证**:上线前凑齐 ≥5 个说得出名字的潜在付费用户(当前 n=2)。

## 二、产品功能(D6–D15)

- **D6 个性化前移**:基础分类过滤(六 category 开关)从 M4 提前至 M2/M3;高级 watchlist(地点圆域、关键词)留 M4。具体筛选交互方式待 arch 设计,原则:「选择即定位」。
- **D7 首启引导**:第一次打开的「你想看什么」引导,按核心界面(前三重要)设计,是垂类分流入口。
- **D8 开屏锚定**:用户关注某地区时,进入即近景该地区(个性化覆盖 SPEC-3.1 默认初始视角)。
- **D9 缓存优先启动**:打开即显示上次缓存事件(球面标记),刷新后呼吸式过渡——部分旧标记熄灭、最新事件亮起。缓存从 SPEC-8.4「可重建不承诺」升格为启动路径的一部分。
- **D10 性能预算**(数字由 arch 提案定稿,预算框架已定):中端手机冷启动→可交互地球 ≤3s;缓存事件上屏 ≤1s;首包 ≤2MB(不含可选纹理);风格包懒加载。
- **D11 事件保留**:默认过期窗口留口子加长(48–72h,视存储定);用户特别收藏的事件永久本地保存;为时间滑块预留缓存窗口,取决于存储需求。
- **D12 地球风格与风格包**:默认风格改为**轻量矢量**(科技感、轻呼吸、静谧);卫星图从默认退役,升级为**气象云图版**(定期从可 fetch 的免费源更新,数据源待调研)进入付费包;像素、艺术风格同为付费。**风格包 ¥6 / $1.99 一口价永久解锁全部(天气/像素/艺术)**。矢量默认风格兼收性能红利(免大纹理)。
  **修订(2026-07-20,依调研 doc/research/market-202607.md,用户拍板)**:一次性解锁在本品类无同行先例,属**待验证商业假设**——上线前安排**最小付费意愿验证**(如 landing page 意向/预售测试,形式由 orch 定),列为发布 gate(与 D5 并列);价格本身维持不变,验证结果可触发 D12 再修订。
- **D13 信源解析分层**(能力越高越晚做):
  - T1 自带坐标源(USGS/EONET/GDACS/LL2)——已 spec,M2;
  - T2 结构化地名 → 本地 GeoNames 精简 gazetteer 查表,M3;
  - T2.5 用户自定义关键词→地点规则(如「Google→硅谷」),纯本地;
  - T3 自由文本地理解析不自研,直接采用 GDELT 地理编码结果;
  - T4 实体→地点智能解析后置为付费能力;早期用**人肉钉图**替代(用户订阅 feed 时自选钉定地点)。
- **D14 自定义 RSS 信源**:归**原生端(Capacitor)功能**——原生请求不受 CORS 限制;网页/PWA 仅支持 CORS 开放的 feed,对不支持者明确提示。内容展示以 **feed 自带内容为限**(feed 含全文则可展示全文),尊重 feed 声明的转载限制,不抓取 feed 之外的正文。
- **D15 AI 能力后置**:摘要/聚类/区域周报等 AI 消化功能不进早期版本,后置为付费功能;前提是数据层(融合/去重/清洗)先做扎实。早期核心 = 解析地点与类型、球面显示、点击查看信源内容(下拉面板 App 端 / 可收起侧栏桌面大屏端)。

## 三、商业与架构(D16–D20)

- **D16 分阶段架构**:
  - 阶段一:零服务器(唯一例外见 D18),收入源 = 风格包一次性解锁;
  - 阶段二:服务器功能(多端同步、真告警、代理 AI、自定义信源代理等)采用**订阅制,价格后定**;
  - **触发条件(定案)**:阶段一净收入月均 ≥ 预估服务器月开销 × 50% 时,启动阶段二 spec 提案。
- **D17 支付通道**:iOS 走 App Store 内购(付费意愿最高的端);其余平台走 Gumroad / Lemon Squeezy 类代销平台发**许可码**,本地校验,跨设备同码,无账号系统。
- **D18 遥测**:匿名心跳 ping——每用户一个随机码,除此不追踪任何个人信息;发布前实现,自订低配服务器。**明示为阶段一唯一服务端组件,不承载任何产品功能**;任何功能上服务器均须过 D16 触发条件闸门。
- **D19 开源策略**:open-core——公开仓库(名为 Worlens)采用 source-available / 非商用方向许可(PolyForm Noncommercial 或同类,最终选型待办);付费风格包与未来服务端闭源。**接受第一个外部 PR 之前必须定稿许可证并配 CLA**。
- **D20 合规调研进路线图**:各免费 API 使用条款核查(CoinGecko 署名要求、OpenSky 匿名额度条款等),确认「客户端直连 + 商用 App」合规;详情卡对第三方内容的展示深度按 D14 口径写入 spec。
