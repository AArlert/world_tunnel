## 一、定位(D1–D5)

- **D1 产品定位**:glanceable「世界的表盘」——为所有想看见世界正在发生什么的人,在碎片瞬间与常显场景,打开三秒看见此刻地球上正在发生什么(且只看自己关心的部分)。不替代信息流,给信息流装一张地图。核心价值 = 感知格局(掌控感、全局观),空间关系本身携带信息。
  **补充(2026-07-21,grill-202607 处理,用户确认)**:体验核心三判据 =**「可信、低噪、为我而变」**——用户爽感的核心来源;后续功能取舍与验收以此三词校准(源:doc/grill/gpt5.6_terra_product_target.md 结论句)。
- **D2 一个产品,N 张面孔**:产品机制通用(自选信源/分类/图层),对外营销按垂类分头打(冲突观察 / 航天 / 灾害等)。目标用户不限于爱好者圈层。
  **修订(2026-07-21,grill-202607 处理,用户拍板)**:首发垂类**按数据能力顺位**——航天与灾害先行(T1 权威源 M2 已落地,可信度天然达标),冲突观察垂类待最小信任层(D22)就位后再主打;机制通用原则不变。
- **D3 界面基调**:宁静。警报感/星球感由用户所选内容决定,App 忠实呈现,但界面本身必须令人平静。球为本体、列表从属(维持 SPEC-2.2 主从关系)。
- **D4 命名**:产品名 **Worlens**(上架前完成商标与 App Store 重名检查——待办);中文 slogan 定案**「看见世界正在发生」**;英文 slogan 待办。仓库内部代号保留 world_tunnel,对外(含公开源码仓库)用 Worlens。
  **修订(2026-07-20,依调研 doc/research/market-202607.md,用户拍板)**:发现在运营产品 WorldLens Live 与 Worlens 近同名且近同构,重名检查由「上架前待办」**前移为近期高优任务**;同步启动备选名征集;名称定稿前对外物料(公开仓库名、slogan 铺开)冻结。
  **二次修订(2026-07-20,命名核查 doc/research/naming-202607.md 呈报后,用户拍板)**:**保留 Worlens**——知情接受与 WorldLens Live 的混淆风险,定位贴合优先,备选名(Worldial/Orbrief/Panorb)留档不启用。名称定稿,对外物料冻结解除,但铺开前须完成:①正式商标检索(美/中+目标市场)与域名 WHOIS 核验;②WorldLens Live 混淆度评估与差异化策略。正式检索若见阻断性冲突,触发本条三次修订。
  **三次修订(2026-07-21,用户反馈)**:对与 WorldLens Live 的混淆顾虑上升,备选名由「留档不启用」升格为**候补序列(推荐排序)**:① **Worldial** ② **Orbrief** ③ **Panorb**(依 doc/research/naming-202607.md Q3:三者搜索零占用、可注册性高、贴合定位),语义向备选 **Stillworld**(直贴 D3「宁静」,娱乐类有撞名)。Worlens 仍为现名;正式商标检索或混淆度评估任一不利、或对外物料铺开前用户改判,即启用候补首位。若 RSS 地理化阅读(D14 修订)升格为主打叙事,命名贴合度按新叙事重估。
- **D5 用户验证**:上线前凑齐 ≥5 个说得出名字的潜在付费用户(当前 n=2)。

## 二、产品功能(D6–D15)

- **D6 个性化前移**:基础分类过滤(六 category 开关)从 M4 提前至 M2/M3;高级 watchlist(地点圆域、关键词)留 M4。具体筛选交互方式待 arch 设计,原则:「选择即定位」。
- **D7 首启引导**:第一次打开的「你想看什么」引导,按核心界面(前三重要)设计,是垂类分流入口。
- **D8 开屏锚定**:用户关注某地区时,进入即近景该地区(个性化覆盖 SPEC-3.1 默认初始视角)。
- **D9 缓存优先启动**:打开即显示上次缓存事件(球面标记),刷新后呼吸式过渡——部分旧标记熄灭、最新事件亮起。缓存从 SPEC-8.4「可重建不承诺」升格为启动路径的一部分。
- **D10 性能预算**(数字由 arch 提案定稿,预算框架已定):中端手机冷启动→可交互地球 ≤3s;缓存事件上屏 ≤1s;首包 ≤2MB(不含可选纹理);风格包懒加载。
- **D11 事件保留**:默认过期窗口留口子加长(48–72h,视存储定);用户特别收藏的事件永久本地保存;为时间滑块预留缓存窗口,取决于存储需求。
- **D12 地球风格与风格包**:默认风格改为**轻量矢量**(科技感、轻呼吸、静谧);卫星图从默认退役,升级为**气象云图版**(定期从可 fetch 的免费源更新,数据源待调研)进入付费包;像素、艺术风格同为付费。**风格包 ¥6 / $1.99 一口价永久解锁全部(天气/像素/艺术)**。矢量默认风格兼收性能红利(免大纹理)。
  **修订(2026-07-20,依调研 doc/research/market-202607.md,用户拍板)**:一次性解锁在本品类无同行先例,属**待验证商业假设**——上线前安排**最小付费意愿验证**(如 landing page 意向/预售测试,形式由 orch 定),列为发布 gate(与 D5 并列);价格本身维持不变,验证结果可触发 D12 再修订。
  **二次修订(2026-07-21,grill-202607 处理,orch 决定)**:对外叙事从「核心价值付费」改为「**Founding Supporter / 支持开发者**」定位——免费版完整可用,付费 = 支持一款不靠广告与订阅焦虑运转的私人世界表盘;价格与包内容不变,纯叙事层调整,并入上条付费意愿验证一并测试文案。
- **D13 信源解析分层**(能力越高越晚做):
  - T1 自带坐标源(USGS/EONET/GDACS/LL2)——已 spec,M2;
  - T2 结构化地名 → 本地 GeoNames 精简 gazetteer 查表,M3;
  - T2.5 用户自定义关键词→地点规则(如「Google→硅谷」),纯本地;
  - T3 自由文本地理解析不自研,直接采用 GDELT 地理编码结果;
  - T4 实体→地点智能解析后置为付费能力;早期用**人肉钉图**替代(用户订阅 feed 时自选钉定地点)。
- **D14 自定义 RSS 信源**:归**原生端(Capacitor)功能**——原生请求不受 CORS 限制;网页/PWA 仅支持 CORS 开放的 feed,对不支持者明确提示。内容展示以 **feed 自带内容为限**(feed 含全文则可展示全文),尊重 feed 声明的转载限制,不抓取 feed 之外的正文。
  **修订(2026-07-21,用户提出)**:新增差异化假设——**RSS 地理化集成**:传统 reader 一律按订阅源组织,Worlens 另提供「按地点在地球上组织」的读法(同一 feed 内条目按地理落点分散上球,如科技媒体蓝点网的欧盟新闻/美国新闻各归其位),与按订阅源视图**并存而非取代**。待 research 竞品核查确认稀缺性(报告落 doc/research/)后,升格为对外叙事与 M3(T2/T3 解析)/M6(原生 RSS)设计输入。
  **修订落定(2026-07-21,依 doc/research/rss-geo-202607.md,orch 决定)**:核查结论**「稀缺但非空白」**——NewsGlobe(newsglobeworldmap.com,站点自述、未实测)已实现「用户自带 RSS + 条目级地理抽取上球」;七款主流/自托管阅读器均无地理组织,World Monitor 等地球类产品为内置源不开放用户 feed。故对外叙事**不得立「首创」**,差异化立足执行层:中文/多语种条目级解析(现有可比产品仅英/欧语源)、零服务器隐私(D16/D18)、信任层(D22)、宁静低噪呈现(D3)。两项硬前置列为 M3 arch 设计输入:① NewsGlobe 对中文 feed(蓝点网类)实测;② **GDELT 中文源落点覆盖度核查**——若不足,D13 T3「直采 GDELT」对中文旗舰用例不成立,须在 arch 设计前裁定。
  **实测补充(2026-07-21,用户实测,截图 doc/attachment/newsglobeworldmap.com.png)**:NewsGlobe 为在营纯网页(2026-07 内容鲜活)、独立开发者署名(Created by DimaRV)、feed 上限 25。弱项:纹理为位图夜景近景严重像素化(反证 D12 矢量默认的正确性);标记颜色编码**信源**而非事件类别,密集区多标记叠罗汉无聚合;信息需两次点击才达信源。强项:「一句话归纳某处发生了什么」的表达方式值得吸收。信息层级教训单列 D23;访问量核查另卡(doc/research/)。
  **硬前置②核查落定+框设更正(2026-07-21,依 doc/research/gdelt-newsglobe-zh-prereq-202607.md,orch 决定)**:GDELT **确实收录中文源**(API 实证`sourcelang:chinese`日均返回、含中文科技媒体域名如 zol.com.cn/itbear.com.cn)且**具专门中文地理编码管线**(GDELT Translingual,标准中文分词+GNS/Wikipedia 跨语言地名)——原措辞「若覆盖不足,T3 对中文旗舰用例不成立」的**因果前提不成立,中文覆盖度本身不是瓶颈**。但同一核查证实**蓝点网(landiannews.com,现主域 www.landian.news)不在 GDELT 索引内**(`domain:` 精确查询返回空),根因是**架构角色错配**:GDELT 地理编码只作用于其自爬语料,**不提供「任意 URL/文本→坐标」的通用服务**,T3 在结构上无法服务「用户自订 feed 按地理落点」这一用例,与该 feed 语言、覆盖度无关。**结论修正**:D13 T3「直采 GDELT」对蓝点网类**用户自订 RSS**旗舰用例不成立**已可定案**(非待覆盖度数据),该用例的地理解析须落 T2(结构化地名,若头条显式地名密度足够)或 T4(付费后置,人肉钉图替代);蓝点网头条初查显式地名稀少、偏机构/产品话题,**倾向指向 T4**——具体走 T2 或 T4、T4 是否需提前于 M3 部分落地,留待 FM-12/13 开卡时 arch 设计裁定,不阻塞本轮 D22 信任层 spec 化。硬前置①(NewsGlobe 中文 feed 交互式实测)本轮**仍未闭合**(调研工具无表单提交能力),留待人工手测或后续轮次。
  **流量核查回填(2026-07-21,依 doc/research/newsglobe-202607.md)**:域名 2026-03 创建(站龄约 4 个月)、流量低于 SimilarWeb/Semrush 测量门槛(StatShow 估 ~30 访问/月)、零商业化、零分发足迹(PH/Reddit/HN/媒体均无)、开发者为俄罗斯独立 web 设计师且未将其收录进本人作品集——定性为**未经分发的 side project / 概念验证**,竞争威胁量级低,「追赶者」名单中权重下调,盯防对象仍为 World Monitor。**注意**:因其从未分发,其低流量**不构成「RSS 上球需求弱」的证据**——需求真伪仍由 D5/D12 发布前验证承担,不得引用本条作需求判断。
- **D15 AI 能力后置**:摘要/聚类/区域周报等 AI 消化功能不进早期版本,后置为付费功能;前提是数据层(融合/去重/清洗)先做扎实。早期核心 = 解析地点与类型、球面显示、点击查看信源内容(下拉面板 App 端 / 可收起侧栏桌面大屏端)。

## 三、商业与架构(D16–D20)

- **D16 分阶段架构**:
  - 阶段一:零服务器(唯一例外见 D18),收入源 = 风格包一次性解锁;
  - 阶段二:服务器功能(多端同步、真告警、代理 AI、自定义信源代理等)采用**订阅制,价格后定**;
  - **触发条件(定案)**:阶段一净收入月均 ≥ 预估服务器月开销 × 50% 时,启动阶段二 spec 提案。
  - **触发条件修订(2026-07-21,grill-202607 处理,orch 决定)**:由单一收入指标改为**联合指标**——留存(如周回访)、活跃用户数、付费转化、预计服务器毛利同时达标才启动阶段二提案;原「净收入 ≥ 月开销 × 50%」降为参考线之一,防服务器成本过低导致过早进入复杂运营。具体阈值随 D12 付费验证结果定稿。
- **D17 支付通道**:iOS 走 App Store 内购(付费意愿最高的端);其余平台走 Gumroad / Lemon Squeezy 类代销平台发**许可码**,本地校验,跨设备同码,无账号系统。
- **D18 遥测**:匿名心跳 ping——每用户一个随机码,除此不追踪任何个人信息;发布前实现,自订低配服务器。**明示为阶段一唯一服务端组件,不承载任何产品功能**;任何功能上服务器均须过 D16 触发条件闸门。
- **D19 开源策略**:open-core——公开仓库(名为 Worlens)采用 source-available / 非商用方向许可(PolyForm Noncommercial 或同类,最终选型待办);付费风格包与未来服务端闭源。**接受第一个外部 PR 之前必须定稿许可证并配 CLA**。
- **D20 合规调研进路线图**:各免费 API 使用条款核查(CoinGecko 署名要求、OpenSky 匿名额度条款等),确认「客户端直连 + 商用 App」合规;详情卡对第三方内容的展示深度按 D14 口径写入 spec。

## 六、信息密度与层级(D23)

日期:2026-07-21 | 来源:用户实测 NewsGlobe 引出的设计问题(「应用应以什么信息密度、层级呈现」)+ orch 讨论

- **D23 信息层级原则「两级半」**(orch 决定,用户可否决):任何事件信息距用户**至多一次应用内点击**。
  - **L0 球面(零点击,注意力预算 3 秒,即 D1「三秒感知」)**:只编码三个变量——**哪里(位置)· 什么类(分类色,SPEC 分类颜色)· 多新(新鲜度,呼应 D9 呼吸式过渡)**;不编码信源(NewsGlobe 反例:信源色让 glance 只回答「谁报的」而非「发生了什么」)。密集区须聚合,不叠罗汉;密度受 D3「宁静」与 D22「少而准」约束。近景/悬停时的一句话归纳交 arch 设计。
  - **L1 详情卡(一次点击,注意力预算 30 秒)**:一屏内给全——一句话归纳 + 信源名与等级(D22)+ 发布/更新时间 + **直接可点的原文链接**;不把「归纳」与「信源」拆成两跳(NewsGlobe 反例)。
  - **L2 原文(外链)**:离开应用的动作,不计入应用内层级。
  - 判据句:**不存在需两次应用内点击才能到达的事件信息**。落 spec 走 §7,作为 M3 详情卡与 L0 标记体系设计(arch)的强制判据。

## 七、视觉分层与审美体系(D24–D29)

日期:2026-07-21 | 来源:orch UX 验收(BUG-022~027,现象截图 doc/attachment/ux-review-20260721/)+ 用户反馈拍板

- **D24 矢量风格视觉分层(LOD)**(用户拍板):
  - **国界要做**;经纬网格默认隐藏(现 SPEC-3.2a 的 30° 网格降级为非默认展示,具体处置随 spec 提案定)。
  - **随相机距离分层显示**:远景不显示国界 → 拉近可见国界 → 再近可见地名;地名密度受控——**大城市固定显示,小城市仅当本地有事件信息时才出现**(密度上限从 D23 L0 与 D3「宁静」推导)。
