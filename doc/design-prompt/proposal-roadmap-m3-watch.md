## 2. 守望前移：M3 最小集 vs M4 精化边界

现状：SPEC-8.1 已把「分类（category 集合）过滤前移至 M2」，「地点（圆域）与主题（关键词）仍为 M4 高级 watchlist」，高级 watchlist = FM-16（M4）。守望闭环要求把「守望命中→呈现」的**最小闭环前移进 M3** 作为脊柱。

### 2.1 M3 守望最小集（新增 M3 FM 行承接）

- **地方**：一处关注地/粗粒度圆域——**直接复用首启引导（SPEC-8.6）已采集的「粗粒度地理关注」维度**（该维度已 spec 化、已驱动开屏锚定），不新增精细圆域交互。
- **人**：R7 定义「一个地方 + 一个名字，纯本地」——本质是带标签的地方守望（地点 + label），薄层。
- **主题**：关键词匹配 title+summary（SPEC-8.1 主题最小形态）。
- **命中呈现**：过滤模式「仅命中」（SPEC-8.2）+ 命中光柱/列表视觉强调（渲染走 §1.2 光柱脊柱）。**命中呈现 = 应用内视觉，不含通知**（通知 = M4，见 §2.3）。

### 2.2 M4 守望精化（FM-16 re-scope）

- 精细圆域（中心 + 半径 km 精确，SPEC-8.1 地点完整形态）。
- 复杂过滤（多条目组合 / 过滤模式完整、SPEC-8.2）与设置面板管理（SPEC-2.4②）。
- 守望持久化完整化（SPEC-8.4）。
- **建议 FM-16 名称由「高级个性化」re-scope 为「守望精化」**，明确其建在 M3 守望最小闭环之上。

### 2.3 边界裁定（须产品拍板，见 §8）

- **守望命中的「通知」归属**：product-vision 守望阶段写「光柱/通知」。arch 判定 **M3 = 光柱（应用内视觉）；通知（SPEC-8.3 Web Notification）= M4/FM-19**（受 R6 通知宪法治理）。理由：真通知需投递机制（Web Notification M4 / Capacitor M6），M3 无 PWA 壳；最小闭环以应用内命中呈现闭合即可。**此边界需 orch/用户确认。**
- **守望前移触及 SPEC-8.1/8.2 里程碑归属**：地点/主题的**最小形态**里程碑归属由 M4 前移 M3、精化留 M4——属对外可见的能力分层，须在 per-R R7 §7 提案里改 SPEC-8.1/8.2 归属句（防两处能力边界含糊）。「人（地点+名字）」为 SPEC-8.1 未有的新守望对象类型，须在 R7 §7 提案里落 spec 正文。

## 3. GDELT / news 火线降选装（R4 + R2）

### 3.2 FM-12 / FM-13 重排（§5.2 防蒸发是硬约束）

FM-12 现 = GDELT + CoinGecko（OpenSky/flight-60s 已于 REV-016/BUG-017 改挂 M6 FM-27）。重排：

- **CoinGecko（R2）→ 删除**：删 SPEC-5.7 条目、SPEC-2.1 ticker 句、SPEC-5.10「CoinGecko…不入本表」注、FM-14 锚点「2.1-ticker」、FM-12 CoinGecko 部分。删除类改动，spec 子句本身移除，无孤儿（无 §5.2 蒸发风险）；须一并 scrub 上述交叉引用（per-R R2 提案负责）。
- **GDELT（R4）→ 后置选装**：SPEC-5.4（GDELT 接口）改挂 defer-target；解析分层 FM-13（SPEC-5.8 T2/T2.5/T3）随 GDELT 一同后移（T2/T3 的唯一 M3 消费者是 GDELT，报道类源移出后 M3 无消费者）。**T1（SPEC-5.8 T1）留 M2/FM-06 不动。**
- **§5.2 防蒸发硬约束**：GDELT 后置为 **defer-target 里程碑的 FM 行**（保留 FM-12/FM-13 承接行），**不得降为「无 FM 行」**——SPEC-5.4/5.8-T2/T3 是真实对外契约（非「不做」），若掉出全部场景则静默蒸发。**里程碑待裁（§8）**，但无论落哪个里程碑，FM 承接行必须保留。

### 3.3 D22 信任层不受 GDELT 后置影响（关键澄清）

- SPEC-5.10 信任分级与 SPEC-5.8 T1–T4 为**正交独立轴**（spec 已 pin）。详情卡信任层（M3-07~10）对 **T1 源即成立**：USGS→「权威事件源」等映射 + L1 展示，T1 源足以覆盖。
- SPEC-5.10 的 **gdelt→「新闻报道（待验证）」映射行留 M3**——M3-07 是纯派生表单测（forward-compat，如 SPEC-5.9 RSS 映射行的先例），live GDELT 缺席不影响其成立，**不构成蒸发**。
- 结论：**D22 信任层详情卡（M3-07~10）照旧 M3 推进**；GDELT 后置只后移 live provider（SPEC-5.4）与解析分层（SPEC-5.8 T2/T3），信任层脊柱不动。
