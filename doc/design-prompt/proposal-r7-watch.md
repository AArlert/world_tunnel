## 1. SPEC-8.1 — 守望对象重铸（三类 + 「人」新类型 + 里程碑分层 + 分类分离）

### 1.1 现行（`doc/spec.md` §8 line 221–222）

```
- **SPEC-8.1** watchlist 三类条目：地点（圆域：中心+半径 km）、主题（关键词，匹配 title+summary）、分类（category 集合）。多条目为「或」关系。
  **分类（category 集合）过滤前移至 M2** 作为首屏基础控件；地点（圆域）与主题（关键词）仍为 M4 高级 watchlist。
```

### 1.2 问题（product-vision §4.4 + REV-018 §3，需求源）

1. **概念词**：「watchlist」是信息产品旧自我的措辞；世界表叙事下对外概念 = **守望**（product-vision §6 R7）。
2. **对象类型不符**：product-vision §4.4 line 117 定守望对象**三类 = 地方 / 人 / 主题**；现行 SPEC-8.1 三类 = 地点 / 主题 / **分类**——缺**「人（一个地方 + 一个名字，纯本地）」新类型**（REV-018 §3 line 118「须落 SPEC-8.1 正文」），且把**分类**混列为守望对象（分类过滤本质是环境显示筛选、非「牵挂之物」，见 §1.4 取舍）。
3. **里程碑归属**：现行「地点/主题仍为 M4」与守望闭环前移 M3 相冲突——REV-018 §3 定「地方/人/主题最小形态 M3、精化 M4」，须重定 SPEC-8.1 归属句（否则守望闭环无 spec 授权）。

### 1.3 建议——SPEC-8.1 重铸文（**加粗 = 本卡新增/改动语义**）

```
- **SPEC-8.1** 守望对象与分类过滤：
  **守望对象三类**（产品对外概念「守望」；代码标识符 `watchlist` 不动）——**地方**（圆域：中心+半径 km）、
  **人**（一处地方 + 一个名字标签；名字纯本地、永不上传，本质为带标签的地方守望）、
  **主题**（关键词，匹配 title+summary）。多守望对象为「或」关系；守望对象驱动守望命中呈现（SPEC-8.2）。
  **里程碑分层**：守望对象的 **M3 最小形态**——地方（复用首启引导 SPEC-8.6 已采集的粗粒度地理关注，
  不新增精细圆域交互）、人（带标签地方守望薄层）、主题（关键词匹配 title+summary）；
  **M4 精化形态**——地方精细圆域（中心+半径 km 精确输入）、主题复杂过滤/多守望条目组合（SPEC-8.2 完整过滤）、
  守望持久化完整化（SPEC-8.4）。
  **分类（category 集合）过滤**为独立的环境显示筛选（六 category 开关，前移 M2，SPEC-2.4①）：
  集合语义（任意子集组合、非互斥单选），全局显隐事件类别；**非守望对象**（不产生守望命中、不进「或」守望集）。
  分类过滤与守望命中为正交筛选轴——分类控类别显隐、守望控命中，二者自然叠加（显示类别内的守望命中），
  不新增复杂组合逻辑（复杂组合属 M4）。
```

### 1.4 取舍陈述——「人」新类型 & 分类分离

- **「人」= 带标签的地方守望（薄层）**：product-vision §4.4「人（一个地方加一个名字，纯本地）」。arch 判定其**本质 = 地点守望 + 一个 label**（非独立坐标解析、非人物追踪），故 M3 即可全量落地、无 M4 精化项（REV-018 §3 line 113「薄层无精化」）。**名字纯本地、永不上传**是对外可见隐私行为（product-vision §4.4「纯本地」+ §9 零服务器），必须进 spec 正文（本卡兑现）。
- **分类分离为「非守望对象」**：product-vision §4.4 守望对象**三类无分类**；且守望的语义是「牵挂之物 → 命中强调/沉默」，分类过滤是「显示哪些类别」的环境筛选（M2 已落六开关）——二者语义分属。故本卡把分类从 SPEC-8.1「守望条目」**剥离为独立显示筛选**、显式标注「非守望对象」。**集合语义（子集组合、非互斥）原样保留在 SPEC-8.1**（M2-19 断言的锚，行为不变，见 §6.2 re-anchor），不蒸发。
- **「或」关系 scope 收窄**：现行「或」跨 地点/主题/分类；重铸后「或」仅跨守望对象（地方/人/主题）。跨类型「或」从未被测（地点/主题原为 M4 未实现），分类改为正交叠加，无 ✅ 判据受影响。

### 1.5 理由

1. **忠于 product-vision**：三类（地方/人/主题）逐字对齐 §4.4 line 117；「人」新类型兑现 REV-018 §3 line 118 义务。
2. **守望前移获 spec 授权**：M3 最小/M4 精化分层句是 FM-28〔守望最小闭环〕的 spec 依据（§5、§6.3），REV-018 §6.4 F-4 要求「FM 占位应用待本句落定」。
3. **极简 · 不为 M4 预写**：M3 最小形态复用已 spec 化的粗粒度地理关注（SPEC-8.6，零新增交互）；精细圆域/复杂过滤只写「M4 精化」名目、不预写正文取值。
4. **零蒸发**：分类集合语义留在 SPEC-8.1、前移 M2 不变；概念词与对象重构不使任何现行 ✅ 子句掉出场景。

## 2. SPEC-8.2 — 过滤模式与守望命中呈现里程碑归属

### 2.1 现行（`doc/spec.md` §8 line 223）

```
- **SPEC-8.2** 过滤模式：全部事件 / 仅 watchlist 命中。命中事件在列表与球面均有视觉强调。
```

### 2.2 建议——SPEC-8.2 重铸文

```
- **SPEC-8.2** 过滤模式与守望命中呈现：
  **M3 最小（守望闭环）**：过滤模式「全部事件 / 仅守望命中」两态；**仅守望命中**模式下仅呈现命中事件——
  命中事件经球面光柱（SPEC-3.7）与事件流列表（SPEC-2.2a）现有渲染呈现、非命中事件隐藏
  （隔离即强调，复用与分类过滤 SPEC-2.4① 同类的显隐机制，不新增视觉轴、不付 D23 字典税）。
  **M4 精化**：完整过滤模式（多守望条目组合逻辑）；全部模式下命中事件的 inline 视觉强调
  （在显示全部事件时额外标示命中项、区分命中/非命中——属新增视觉轴，取值另由 aes 方案 + arch 落 SPEC-8.2，
  比照 SPEC-3.7 光柱先例）。
```

### 2.3 取舍陈述——命中强调的「隔离式（M3）vs inline（M4）」

- **arch 裁量**：M3 最小闭环的「命中强调」采**隔离式**——「仅守望命中」模式下命中事件独占呈现（复用光柱/列表 + 与分类过滤同类的显隐机制），**零新增视觉轴、零字典税**。这即 REV-018「仅命中最小模式 + 命中强调 → M3」的最小落地读法：命中事件经既有光柱/列表渲染、隔离即是强调。
- **inline 命中强调后置 M4**：在「全部」模式下额外标示命中项（区分命中/非命中）需**新增视觉轴（字典税，D23）**——是锦上添花、非闭环必需，后置 M4 避免 M3 提前付字典税、避免 FM-28 背 aes visual-pin 前置依赖。
- **歧义待裁**：若 rev 认为 REV-018「命中强调」必含 inline 区分于 M3，则 FM-28 需前置 aes 方案（命中态光柱/列表编码取值）→ arch 落 SPEC-8.2，类比 FM-07 光柱待动效批。列入 §7 歧义清单，不默选。

### 2.4 理由

1. **honor REV-018 + 极简**：两态过滤 + 隔离式命中呈现完整闭合守望闭环（引导采集→本地存→命中→仅命中过滤），是最小可闭环边界（任务硬约束「仅命中过滤即为闭环候选边界」）。
2. **字典税纪律（D23）**：inline 命中区分是新增视觉轴，交 aes 付税裁决、不在 M3 擅立。
3. **零蒸发**：现行「命中事件在列表与球面均有视觉强调」子句拆为 M3 隔离式 + M4 inline，两半均在 spec、各有 FM 承接行（FM-28/FM-16），无子句掉出场景。
