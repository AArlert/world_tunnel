## 1. SPEC-2.2 — 表盘默认态重铸 + 事件流面板退位「今日刻痕」抽屉（R3）

### 1.1 现行（`doc/spec.md` line 40）

```
- **SPEC-2.2** 主区：全屏地球 canvas；右侧悬浮事件流面板（宽 300px，可折叠）。**面板底亮度不得高于夜半球底面（SPEC-3.2a 夜端 `#0d1827`），使面板退后、球为本体（D3「球为本体、列表从属」）；面板具体底色/边框/圆角/质感（实底或玻璃）属 UI 实现自由度。**
```

### 1.2 问题（product-vision §4.3 R3 + REV-018，需求源）

1. **常驻面板与静谧默认态相杀**：常驻右侧 300px 面板是「一条常驻叫嚷的事件流」，与「你守望的世界安好」相杀（product-vision §4.1 line 95、§6 R3 line 139）；世界表默认态应是「球 + 时间 + 光柱，零 chrome」（§4.3 line 111）——静谧默认态是安好态（R5）的背景板（REV-018 §1.4 R3 判定）。
2. **「事件流」是 feed 旧自我措辞**：世界表叙事下，退位后的抽屉 = **「今日刻痕」**（product-vision §6 R3 line 139；「刻痕」词根见 §3 line 56）。
3. **默认展开态无 spec 授权收起**：现行「宽 300px，可折叠」+ M2-13 断言「初始宽度 300px（默认展开）」——须重定默认态为**默认收起**，否则守望闭环无静谧舞台。

### 1.3 建议——SPEC-2.2 重铸文（**加粗 = 本卡新增/改动语义**）

```
- **SPEC-2.2** 主区与表盘默认态：主区为全屏地球 canvas，占满视口宽高。
  **表盘默认态（冷启动静息态）以地球为视觉主体，静息核心 = 地球 canvas + 事件光柱标记（SPEC-3.7）+ 顶栏 UTC 时钟（SPEC-2.1「时间」）**；
  **事件流面板不常驻画面，退位为默认收起的「今日刻痕」抽屉**（「球为本体、列表从属」D3 走到底：静息即球、时间、光柱，静而克制）。
  抽屉冷启动即收起态，提供展开/收起双向切换控件；展开后呈现事件流列表（SPEC-2.2a 列表行），
  沿用现有事件流面板形态——展开态悬浮于 canvas 之上、不挤占 canvas，展开宽度约 300px。
  抽屉展开态底亮度不得高于夜半球底面（SPEC-3.2a 夜端 `#0d1827`），使抽屉退后、球为本体（D3）。
  抽屉对外名「今日刻痕」（product-vision 基准，最终措辞属 aes 制定项）；抽屉展开态具体底色/边框/圆角/质感（实底或玻璃）、
  收起态触发控件的形态与位置、展开/收起动效属 UI 实现自由度，其中视觉呈现取值交 aes 制定（后续 /aes-review 批次）。
  **本条重定义仅及事件流面板（→今日刻痕抽屉）与表盘默认态**；顶栏（SPEC-2.1）与基础分类过滤六开关（SPEC-2.4① 首屏可见控件）的首屏呈现不由本条改动。
```

### 1.4 取舍陈述

- **「今日刻痕」命名**：product-vision §6 R3 基准。arch 采为抽屉对外名入 spec 正文，最终措辞交 aes 微调（D25 先例）。现有 EventPanel「事件流」标题（line 64）/折叠按钮文案（line 55）→「今日刻痕」（DEV 实现层）。**命名一致性注**：「今日刻痕」（今日事件抽屉，R3）与「刻痕/收藏」（star 永久保存，§3 line 56 / R12/M4）词根共用、相关但分属——非本批阻断，列 §5.2 遗留 1 交 aes/后续叙事。
- **展开宽度沿用约 300px**：product-vision R3「常驻右侧 300px 面板 → 默认收起的抽屉」保留 300px 参照。arch 保留为**展开态宽度**（沿用现值、避免蒸发、不新造值）；抽屉几何微调交 aes（§3）。
- **面板底亮度 cap 保留**：现行 cap 句（M3-05 锚点）平移为**抽屉展开态**约束，语义不变、不动取值。
- **「零 chrome」精确化为「无常驻事件流面板」**：spec 正文不写不受约束的绝对「零 chrome」（会与 SPEC-2.4① 六开关首屏可见相撞），改写为「静息核心 = 球+光柱+钟 + 事件流面板退位」，并显式声明本条不改顶栏/六开关呈现——精确、无 spec 内部矛盾。六开关是否随「零 chrome」一并退位属独立更大改动（触 SPEC-2.4①/M2-19/FM-10），列 §5.1 歧义 1 待裁、不默选。

### 1.5 理由

1. **忠于 product-vision R3**：默认态「球+时间+光柱、事件流退位」逐字对齐 §4.3 line 111 / §6 line 139；「今日刻痕」命名兑现同行。
2. **静谧默认态是安好态背景板**：REV-018 §1.4「静谧默认态是安好态的背景板」判定的 spec 落点。
3. **极简 · 抽屉复用**：EventPanel 已有 collapsed 状态机，R3 = 翻默认态 + 重命名 + 改叙事，零重设计（product-vision「已实现代码不废」）。
4. **零蒸发**：300px（→展开态宽度）、折叠双向切换（→默认收起翻转）、canvas 占满视口不挤占、列表三要素、排序均保留在 spec；空状态子句交 R5（§2）。防蒸发逐子句核见 §4.2 M2-13 条。

## 2. SPEC-2.2a — 安好态取代空状态（R5，空状态子句重定义）

### 2.1 现行（`doc/spec.md` line 41 末端相关子句）

```
… 无事件时显示空状态文案「暂无事件」。 …
```

（SPEC-2.2a 其余子句——三要素/severity 编码/排序——本卡**不动**，仅改此一空状态子句。）

### 2.2 问题（product-vision §4.1 R5 + REV-018 §5-C，需求源）

1. **「暂无事件」是道歉**：仿佛无事发生是产品失职（product-vision §1 line 25、§4.1 line 95）。世界表叙事：**沉默即工作**，安好态是旗舰功能——「敢说这句话是靠停留时长活着的产品结构上做不到的护城河」（§6 R5 line 141）。
2. **空状态无守望语境**：「暂无事件」不区分「**有守望对象且无动静**」（安好）与「**无守望对象**」（未设置守望）。R7 已立守望对象有/无载体（SPEC-8.1/8.6），R5 承接触发判定细化——REV-018 §5-C / roadmap §8-C 明令「避免对无守望用户说『你守望的世界』」（line 127 / line 217）。

### 2.3 触发条件设计（依托 SPEC-8.1/8.6 守望对象载体，F-C 义务）

事件流列表（今日刻痕抽屉展开态，经全部生效过滤后）**呈现集为空**时，依守望对象有无二分：

| 空态类型 | 触发条件（守望对象载体 = SPEC-8.1 三类 / SPEC-8.6 采集） | 呈现 | 依据 |
| --- | --- | --- | --- |
| **安好态** | 有 ≥1 守望对象 **且** 列表呈现集为空（**典型触发 = 「仅守望命中」模式零命中**，SPEC-8.2；亦含「全部事件」模式零事件） | 文案「**你守望的世界，此刻安好**」（product-vision §4.1 基准句，承接 SPEC-8.1 守望对象 / SPEC-1 世界表词根） | product-vision §4.1 line 95、§6 R5 line 141 |
| **无守望对象空态** | 无守望对象（引导跳过/未设置，SPEC-8.6「可跳过」）**且** 列表空 | **不预设守望叙事的中性空态**（不套用「你守望的世界」，可含指向守望设置 SPEC-8.6 的引导；**文案本体属 aes 制定项**） | REV-018 §5-C（line 127）、roadmap §8-C（line 217） |
| 搜索无匹配 | SPEC-2.5 搜索无命中（**独立查询语境**，非静息安好态） | 「当前缓存内无匹配」（SPEC-2.5 现句，**不改**） | SPEC-2.5（line 47） |

### 2.4 取舍陈述

- **安好态 vs 无守望对象空态二分**：product-vision 基准句「你守望的世界」预设有守望对象；对未设置守望的用户说会说谎。故按守望对象有/无二分——守望对象载体 = R7 已落 SPEC-8.1（三类）/SPEC-8.6（采集/可跳过/未选回落）。这是 REV-018 §5-C「F-C 触发判定归 R5」的兑现。
- **无守望对象文案本体 = aes 制定项**：product-vision 只给「你守望的世界，此刻安好」（有守望对象 case），未给无守望对象文案。arch **不新造文案**（D25：aes 方案→arch 落 spec）——**语义决策（中性、不套守望叙事、可引导 hook）入 spec 正文**（对外行为），文案本体交 aes（措辞）。此拆分与 SPEC-8.6「引导内呈现权重属实现自由度」同构，非行为泄漏。
- **安好态文案本体入 spec 正文**：「你守望的世界，此刻安好」是 product-vision 基准（非 arch 新造），入 spec 正文；最终措辞 aes 可微调（比照 SPEC-2.2a 已有「格式细节属实现自由度」的留白口径）。
- **搜索无匹配保持独立**：SPEC-2.5 搜索是显式查询语境，非静息安好态；「当前缓存内无匹配」不改、不并入安好态（避免对搜索空结果说「你守望的世界安好」）。
- **「暂无事件」删除**：旧道歉式措辞删除、无残留（`grep 暂无事件` 全库仅 SPEC-2.2a + M2-13 + EventPanel.tsx，均随本批更新）。

### 2.5 建议——SPEC-2.2a 修改文（仅替换末端空状态子句，**其余子句保持不变**）

**原子句**（删）：

```
无事件时显示空状态文案「暂无事件」。
```

**新子句**（入）：

```
事件流列表呈现集为空时的空状态依守望对象有无（SPEC-8.1/8.6）二分呈现：
**有 ≥1 守望对象时显示安好态文案「你守望的世界，此刻安好」**（世界表叙事，承接 SPEC-8.1 守望对象与 SPEC-1 世界表词根；
product-vision 基准，最终措辞 aes 可微调）——表达「你守望之物此刻无动静、沉默即工作」，非「暂无事件」式道歉，
典型触发为「仅守望命中」模式零命中（SPEC-8.2）；
**无守望对象时（引导跳过/未设置任何守望对象，SPEC-8.6「可跳过」）显示不预设守望叙事的中性空态**
（不套用「你守望的世界」，可含指向守望设置 SPEC-8.6 的引导；具体文案本体属 aes 制定项，后续 /aes-review 批次）。
搜索无匹配的空态由 SPEC-2.5 独立承接（「当前缓存内无匹配」），不并入本条安好态。
```

### 2.6 理由

1. **兑现 R5 护城河句**：「你守望的世界，此刻安好」逐字入 spec，承接 SPEC-1/SPEC-8.1 已立守望词根，事实源自洽。
2. **F-C 触发判定落地**：REV-018 §5-C 把安好态触发条件归 R5 细化——本卡依 SPEC-8.1/8.6 守望对象有/无载体给出二分触发，无悬空。
3. **零蒸发 · 外科手术**：仅替换空状态一子句，SPEC-2.2a 三要素/severity 编码/排序子句原样保留（M3-01/M2-13 排序断言不受影响，见 §4.2）；「暂无事件」承接点（M2-13 空状态子句）转为安好态二分承接、不掉出场景。

## 5. 歧义清单 & 遗留风险

### 5.1 歧义清单（待 rev 仲裁，不默选）

1. **【歧义·「零 chrome」与六开关首屏可见】** product-vision R3「默认态零 chrome」vs SPEC-2.4①「六 category 开关首屏可见控件」——**R3 是否要求六开关也退位（进抽屉/隐藏）？**
   - **读法 A（arch 倾向，极简）**：R3 scope 限于事件流面板（SPEC-2.2）；六开关保持首屏可见（SPEC-2.4① 不动，M2-19 判据不变、仅机位调整）。「零 chrome」在 spec 正文精确化为「无常驻事件流面板」，并显式声明本条不改顶栏/六开关呈现（§1.3 末句）——避免与 SPEC-2.4① 相撞。
   - **读法 B（强）**：「零 chrome」要求六开关也退位——触 SPEC-2.4①/M2-19（降级/re-scope）/FM-10，改面更大、属超 R3 SPEC-2.2 scope、非 R3 必需。
   - **arch 倾向 A**（product-vision R3「动的是什么」列仅 SPEC-2.2；六开关退位是独立变更、不夹带本批）。**交 rev 定读法**；若 B，则 SPEC-2.4① + M2-19 + FM-10 涟漪须另案补齐，本卡 SPEC-2.2 末句相应调整。

2. **【歧义·安好态文案在默认收起态的呈现】** R3 使今日刻痕抽屉默认收起，安好态文案（SPEC-2.2a 列表空状态）仅在**抽屉展开时**可见；而 product-vision §3「它的一天」把安好（无光柱球面）表达在默认视图/锁屏——**安好态是否须在默认收起态也有可见呈现？**
   - **读法 A（arch 倾向）**：R5 scope 限于 SPEC-2.2a 列表空状态文案（任务 scope）；默认视图的安好由 R3 静谧球面（无守望命中→无光柱）承载；收起态额外安好呈现属 aes/实现自由度，非 R5 硬性。
   - **读法 B**：安好态文案须在默认收起态可见（强化旗舰句）——须定收起态呈现载体（对外可见行为），升 SPEC-2.2/2.2a 正文。
   - **arch 倾向 A**（任务 scope = SPEC-2.2a 空状态；不越界造收起态新对外行为）。**交 rev**；若 B，收起态安好载体入 spec 正文（取值仍交 aes）。

3. **【歧义·分类全关导致的空列表是否套安好态】** 有守望对象 + 六开关全关 → 列表空（M3-12：分类关闭后即使命中守望也不呈现）。
   - **读法 A（arch 倾向，极简）**：套安好态（有守望对象 + 列表空）——叙事不假（你守望之物此刻无呈现），不新增特例。
   - **读法 B**：分类全关是用户显式收窄，应显区隔提示（如「已隐藏全部类别」）而非安好态——避免误导「以为世界安好实为自己关了显示」。
   - **arch 倾向 A**（极简、不新增特例；§2.3 触发按「列表呈现集为空」统一）。**交 rev**；若 B，SPEC-2.2a 须补分类全关空态特例（文案本体 aes 制定项）。
