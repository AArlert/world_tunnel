## 0. 一句话方案与 signature

把「一颗均匀深蓝球 + 满屏同权红点」重构为**一颗被真实阳光照亮、大部分安静、只有真正重要的事在发光的地球**。

- **本批唯一 signature = 昼夜晨昏梯度(the living terminator)**:昼半球被抬亮、随离日角度柔和衰减,夜半球沉入近太空的深靛;跨晨昏线不只是变暗,还有一次克制的**冷色相位移**(昼侧偏青、夜侧偏靛)。这是「世界的表盘」概念的字面兑现,也是 cobe「定向光 + 柔和光照」方法论的直接借用。
- **其余一律克制**:经纬网默认隐藏、海岸线降饱和退为结构线、面板与球同流派、标记按 severity 分轻重。
- **交付前已「减一件」**:见 §5,初稿的三处叠加(标记透明度通道、列表字重通道、大气昼夜差)在修订中各砍到最小或标为可选。

> 现状为什么不达标(量测证据,非「感觉」):对 `initial-atlantic.png` 球体内部采样(排除海岸线/标记的亮像素),左右半球底面平均亮度 Y=10.2 vs 11.3,**比值 1.11:1 ≈ 完全均匀**——没有任何半球级明暗,正是 D24 点名的「整球均匀深色」。同时六类事件里数量占绝对多数的 M2.5–4.5 小震(severity 1)与 M5+ 强震(severity 3)在球面上是**同一支高饱和红、同一量级光环**,即黑名单#1 Dark Acid Default 的 3D 版。

一切层次挂在这条「从太空到最亮小元素」的亮度阶梯上。**结构层(空间/球/大气/海岸)用冷蓝,信息层(标记/标题)才允许高饱和与高亮**——信息永远比结构响。

| --- | --- | --- | --- | --- | --- |

## 初稿(Pass 1)

### 3.1 球体配色层次(判据 1)

经纬网默认隐藏后,默认远景画面的构成 = **太空 → 大气边缘 → 昼夜底面梯度 → 海岸线 → 标记**,五层,无网格。网格拿掉后昼半球容易「空」,靠 §3.2 的离日衰减梯度填满光影,不靠网格线。

- **底面**:改 SPEC-3.2a 现「统一 `#0a1a2f`、夜端由亮度系数压暗」为**两端显式 pin**——昼端(次日点)`#1f4468`、夜端 `#0d1827`。两端 hue 分别 ≈209°/217°,插值时自然完成「昼青→夜靛」的冷色位移。陆地仍不单独填色(维持 SPEC-3.2a 裁定),只由海岸线勾。
- **海岸线**:昼端由 `#4db8ff`(高饱和青、Y≈189)改 `#6690b3`(S≈34%、Y≈138)。理由:现值与 news 分类色 `#40a9ff` 几乎同一支蓝,海岸线会被误读成新闻标记;降饱和后读作「结构」。夜端海岸沉为 `#3a5a72` 低强度自发光,仅勾陆廓。
- **经纬网格**:默认**隐藏**(D24)。保留 `#1e3a5f` 仅供近景 LOD/调试档启用(具体显隐规则属第二批)。
- **大气**:昼侧沿用 `#4a90d9` 菲涅尔边缘;新增**昼夜边缘差**——夜侧压到冷暗地板 `#20344f`(Y≈50,仍高于太空,保证球缘任何角度都勾得出),强度随 `sunDir·N` 调制。不遮标记(SPEC-3.4 不变)。
- **星空**:SPEC-3.5 数量/球壳不变;新增**亮度分布**——绝大多数星 Y 20–45(几乎隐),约 8% 亮星 Y 60–120,**上限 ≤120**(永不超过昼面海岸/标记)。色近中性冷白 `#c8d2dc`,微冷。目的:给深度、避免单调,但绝不成为噪声。

### 3.2 昼夜半球柔和对比(判据 2)

实现参照:**cobe 式定向光 + Lambert/Phong 柔和衰减**(不是硬两值混合)。一盏方向光代表太阳,昼半球底面亮度随 `t = dot(N, sunDir)` 做**平滑余弦式**衰减(次日点最亮 → 近晨昏线渐暗 → 过渡带 → 夜半球地板),而非现状的「一块昼色 + 一块夜色平涂」。

量化「昼夜可辨且柔和」——两条可截图量测的判据:

- **主判据(半球可辨)**:取晨昏线两侧**等角距(建议各 40°、同纬度)**的两个底面采样点,**昼侧 : 夜侧亮度比 ∈ [1.8, 2.6]**。
  - < 1.8 = 单调/不可辨(**现状 1.11,严重不达标**);
  - > 2.6 = 过硬、像探照灯,失「宁静」。
  - 建议两端色(`#1f4468`/`#0d1827`)+ 余弦衰减调到落在此带内(次日点峰值比 2.76 是峰值,代表性昼点会低于峰值、落入带内);由 dev 采样验证、微调衰减曲线命中。
- **副判据(是被照亮的球,不是平板圆盘)**:昼半球**内部**须有梯度——次日点亮度 : 昼侧近晨昏线亮度 **≥ 1.3:1**。这是网格拿掉后防「空/单调」的关键。
- **晨昏过渡带**:全宽建议 11°–17°(SPEC-3.2① 现 t∈[-0.1,+0.1]≈11° 可保留或略放宽)。**这是品质细节,不是目标**(D24 明示:半球级对比才是目标,晨昏锐利与否非关键)。

### 3.3 列表行 severity 编码(判据 3 · 填 REV-012 §5.3 留白)

约束(REV-012 §5.3):3 档、单调、**仅非色相通道**、色相仅表分类、severity 数值/文字/徽章不作显式行元素、不变量 B(视觉突出度随 severity 单调 sev3>sev1)。

**承载通道选定**:

- **主通道 = 标题文本明度(value/opacity)**。这是最强、最可读、最贴 Apple Maps 排版层级的通道;明度/透明度是 REV-012 明列的非色相通道。
  - sev3 标题 `#eef2f7`(近白,对面板对比 16.2:1)
  - sev2 标题 `#c2ccd8`(浅灰蓝,11.2:1)
  - sev1 标题 `#8794a3`(哑灰蓝,5.9:1,仍过 AA)
  - 单调递增、全部可读(最低档 sev1 仍 > 4.5:1 AA)。
- **一致性(非独立通道)= 行首圆点**沿用球面同一枚标记的 severity 色变换(§3.4),使「列表行 ≡ 球面标记的镜像」(D3)。它承载色相(分类)+ 明度/饱和(severity),与主通道同向。
- **辅助通道(可选)= 字重**:仅 sev3 抬到 600,其余 400;两态微推,不做四通道堆叠。
- **不用行高/行密度编码 severity**:会让列表参差、伤扫读——保持行高统一。
- 次要文本梯度(全档统一,不随 severity):相对时间 `#6d7a8a`(4.2:1,始终最弱)、面板头「事件流」`#93a1b0`。

不变量自查:A(同 category 跨 severity 色相恒定)——行首点只做明度/饱和变换,hue 不动 ✓;B(单调)——标题 `#eef2f7 > #c2ccd8 > #8794a3` 明度单调 ✓;C(无 severity 数值/文字/徽章)——只改文本明度/字重,不加可读元素 ✓。

### 3.4 事件标记色在深底上的 severity 层次(判据 4)

分类色相**不动**(SPEC-3.7 语义)。在**明度/饱和度/发光强度**(spec 明确此三者属审美范围)上按 severity 分层,让 sev3 有权重、sev1 退后,破「满屏同权高饱和点」。SPEC-3.7 已用尺寸/脉冲编码 severity;本方案**叠加**明度/饱和/辉光,冗余编码更可读。

变换规则(以 HSL 乘子表达,sev3 = SPEC-3.7 pin 色本身):

| 档 | 饱和 S× | 明度 L× | 辉光 | 透明度 |
| --- | --- | --- | --- | --- |
| sev3 | 1.00 | 1.00 | 持续脉冲光环(bloom) | 1.0 |
| sev2 | 0.82 | 0.93 | 静态柔光环、无脉冲 | 1.0 |
| sev1 | 0.60 | 0.82 | 无辉光 | 0.85 |

据此算出六类分级色(arch 可直接引用;若调乘子,按规则重算):

| category | sev3(=SPEC-3.7) | sev2 | sev1 |
| --- | --- | --- | --- |
| disaster | `#ff4d4f` | `#ed484a` | `#cf4142` |
| conflict | `#ff7a45` | `#ec7241` | `#ce653c` |
| humanitarian | `#ffc53d` | `#ecb73a` | `#cda036` |
| news | `#40a9ff` | `#3d9dec` | `#388acd` |
| launch | `#b37feb` | `#a674dc` | `#9366c3` |
| flight | `#5cdbd3` | `#57cac3` | `#4fb0aa` |

最大的权重杠杆是**辉光**:sev3 会发光(bloom)、sev1 是一枚哑色小点。加州/阿拉斯加的小震簇于是变成「一片安静的哑红点 + 偶尔一颗发光的强震」,而非现状一片同权亮红。

## 4. 逐条自我批评(Pass 2)

### 4.1 对照反模式黑名单(7 条)

1. **Dark Acid Default(近黑底+单一高饱和点满屏)** —— 初稿是否真破了?底盘已建 8 阶明度梯度(不是「一深底」),标记已 severity 分层(不是「同权高饱和」)。**破了**。但需自查:sev1 的「降饱+降明+缩小+去辉光+降透明」= 5 项同时削,密集簇里 sev1 会不会**掉到看不见,丢事件**?→ 真实风险,见 §5 修订。
2. **紫/靛渐变** —— 结构层 hue 锁 200–225°,夜底 217° 是蓝;唯一的紫是 launch 语义小标记,非渐变背景。**未触**。但需加护栏文字,防实现漂到 250°+。
3. **玻璃拟态+霓虹辉光当装饰** —— 面板实底、无模糊玻璃;唯一辉光是 sev3 标记 bloom,解决「哪个最重要/最新」的真实层叠问题,非装饰。**未触**。
4. **圆角堆砌 / 0.1 阴影** —— 面板用 1px 发丝边不用重阴影;行不做卡片、圆角 ≤6px。**未触**。
5. **均匀无层次配色** —— §2 明度阶梯即针对此。**破了**。
6. **无个性默认字体(Inter/Roboto)** —— 本批是配色,不定字体;§3.3 只引「字重」抽象值,未指定字族。**不越界**(字体属另一批,见移交)。
7. **逢 hover 弹跳/常驻装饰动画** —— sev3 持续脉冲(SPEC-3.7 强制)是否算「常驻装饰」?若满屏 sev3 都在脉冲,确有装饰化风险。→ 见 §5 修订(脉冲耦合新鲜度)。

### 4.2 对照 3D 专属判据(5 条)

- **半球明暗对比** —— §3.2 给了可量测双判据 + 现状 1.11 的反证。达标路径清楚 ✓。
- **深底上的事件色层次** —— §3.4 用非色相三通道分层,不动色相语义 ✓。
- **注记/标记密度随相机分层(LOD)** —— 标记的 LOD 显隐属第二批;但 severity recede + SPEC-5.0a 门槛已先削一部分密度。本批不欠账、但需注明「密度分层待第二批」。
- **临边(limb)观感** —— 大气昼夜差可能把夜侧球缘压没(丢球体轮廓)?→ §5 给夜侧地板下限。海岸线在球缘透视压扁易与大气混色 → 需补「近球缘海岸线淡出」建议。
- **面板与球图底关系** —— 面板底 `#0c1622` 不亮于夜半球底、与球同族、非玻璃 → 不平摊盖信息、同流派 ✓。

### 4.3 对照品味原则

- **一个 signature 其余克制** —— signature = 昼夜晨昏梯度(唯一让人记住的)。但初稿另塞了三处「加法」:标记透明度通道、列表字重通道、大气昼夜差。**不够克制,须减一件(其实要减三件)**。
- **层次靠明度/饱和度不靠加颜色** —— 已遵守:全部层次在蓝色相族内用明度/饱和;唯一色相是 6 类语义色。✓
- **主色 + 锐利点缀** —— 冷蓝主场 + sev3 标记锐利发光点缀,拒 timid 均匀。✓

## 5. 修订(Pass 2 → delta,标出改了什么、为什么)

1. **标记 sev1 去掉透明度通道**(初稿 0.85 → **改回 1.0**)。理由:5 项同时削会把 sev1 削到近隐、可能丢事件;severity 分层的目的是「安静」不是「消失」。保留降饱/降明/缩小/去辉光四项已足够 recede,透明度这一件**减掉**。→ §3.4 表 sev1 透明度列作废,统一 1.0。
2. **列表字重降为「可选」而非必需**(减一件)。主通道锁定在标题明度;字重只作 dev 可选微增强,不进硬判据。避免「明度+字重+点色」三通道在一行里堆成噪声。
3. **大气昼夜差标为「可选 signature 候选,可整体砍」**。若实现/验收觉得多,直接退回 SPEC-3.4 单色均匀大气,不伤主 signature(昼夜梯度在球面本体已经成立)。**但若保留,夜侧大气须给亮度下限**(地板 `#20344f`,Y≈50 > 太空 9),确保任何相机角度球缘都勾得出,不被压没(补 §4.2 临边风险)。
4. **sev3 脉冲耦合新鲜度**(回应黑名单#7):建议脉冲幅度 = f(severity 高 **且** 事件新鲜度),随事件变旧而衰减,而非永久常驻——让动效服务「新事件」信息(呼应 D9 呼吸式),而非装饰。**注意 SPEC-3.7 现文「severity 3 必须有持续脉冲环」是硬 spec,本项为向 arch 的提案式细化(脉冲存在但幅度随龄衰减),不覆盖 spec**;并须尊重 reduced-motion(P1 非协商底线)。
5. **补「近球缘海岸线淡出」**:贴球面的海岸线在 limb 因透视压扁、易与大气辉光叠加混色,建议海岸线不透明度随 `|N·view|` 在球缘淡出,避免露怯(临边判据)。
6. **补色相护栏文字**:结构层 hue 实现须 clamp 在 200–225°,任何情况不得漂入 250°+(防黑名单#2)。

## 7. 被否方案与理由

| 被否方案 | 为何否 |
| --- | --- |
| 晨昏带走「暖→冷」真实黄昏色(昼侧偏暖橙/金) | 暖橙与 disaster 红标记撞色、读作 acid;要暖只能经紫(黑名单#2)。改走「昼青→夜靛」纯冷位移,同样不单调、无撞色。 |
| 夜半球压到与太空同亮(真实「地球暗面消失」) | 好看但夜侧事件/海岸全丢,违背新闻球「夜侧也要看见」。改为夜底仅比太空高一档 + 大气勾缘。 |
| severity 用**行高/行密度**编码 | 列表参差、伤扫读,且易被读成分组而非轻重。改用标题明度(排版层级)。 |
| 列表 severity 叠「明度+字重+点色+行高」四通道 | 过度编码=噪声,违「减一件」。收敛到主通道(标题明度)+ 镜像点,字重可选。 |
| 用「加一种新颜色」区分 severity(如高危加描边红) | 违 SPEC-3.7 色相语义 + REV-012 非色相约束 + 黑名单#5;层次只准靠明度/饱和/发光。 |
| 提亮整球底面到「一眼很亮的地球」 | 违 D3 宁静、丢深色流派;抬亮只给昼半球、且是「柔和抬」不是「打探照灯」(判据上限 2.6)。 |
| 海岸线保持高饱和青 `#4db8ff` | 与 news 分类蓝 `#40a9ff` 几乎同支,结构线被误读为事件。降饱和解决。 |

## 8. 移交项(需 arch 落 spec 的取值 / 需 orch 或第二批处理)

**需 arch 提案落 spec(§7):**

1. **SPEC-3.2a**:底面两端显式 pin(昼 `#1f4468` / 夜 `#0d1827`);海岸线昼端改 `#6690b3`、夜辉光改 `#3a5a72`;经纬网默认隐藏(现「每 30°」降级为非默认);夜端插值改「显式两端 + 余弦式衰减」以承载 §3.2 半球对比判据。
2. **SPEC-3.2① / 新判据**:昼夜半球对比的**对外可验收判据**(主判据比值 [1.8,2.6]、副判据 ≥1.3:1)——这是对外可见质量契约,建议入 spec 或 testplan 判据,不能只留本文件。
3. **SPEC-3.4**:大气夜侧地板 `#20344f` + 强度随 `sunDir·N`(**标为可选**;若采纳则连同「近球缘海岸线淡出」一并)。
4. **SPEC-3.5**:星空亮度分布与上限(≤120)。
5. **SPEC-3.7**:severity 的明度/饱和/发光分层规则(§3.4 乘子表 + 六类分级值);建议同时把「sev3 脉冲幅度随新鲜度衰减」作为提案细化(**注意与现「必须持续脉冲环」文字的关系,由 rev 仲裁**)。

## 9. 验证方法(达标怎么量)

- **昼夜对比**:dev/qa 起 `make dev`,同视角截图,采样晨昏线两侧 ±40° 同纬度底面点(排除 Y>阈值 的海岸/标记像素),算 sRGB 亮度比,须 ∈ [1.8,2.6];昼半球内部次日点:近晨昏点 ≥1.3:1。**验收对照基线 = 现状 1.11:1(本文件已量测)**。
- **标记层次**:同屏 sev1/sev3 标记,sev3 有 bloom、sev1 无;密集簇「多哑点少亮点」不再满屏同权。
- **列表 severity**:同屏三档,标题明度单调 sev3>sev1,全档 ≥ 各自 WCAG 目标(标题 ≥4.5:1、次要 ≥3:1)。
- 以上均为 AES 验收职 `/aes-review` 的取证依据。
