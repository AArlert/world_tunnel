## 0. 问题陈述（两缺陷根因）

### 0.1 BUG-023 —— 无门槛的噪声淹没

`src/data/providers/usgs.ts` 接 USGS `all_hour`/`all_day` **全量** feed 且归一化不设任何震级下限（`normalizeFeature` 只校验字段存在性）。实测快照 250+ 条中**绝大多数为 M<2 仪器级微震**（多为加州地热田、内华达同点位重复），球面一屏 200+ 红点、事件流被淹没，M5.6 有感地震与 M0.35 微震**同屏同权**。直接违背：

- **D1**「三秒看见此刻地球上正在发生什么」的**低噪**核心；
- **D23 L0**「只编码三变量、密集区须聚合、**少而准**」；D22「少而准」。

根因：**SPEC-5.1 只 pin 了 severity 映射，未 pin 任何呈现门槛**；且全 §5 无「哪些归一化事件才应被呈现」的跨源契约——即噪声准入是**规格空白**，非实现 bug。

### 0.2 BUG-025 —— 列表行无轻重，低价值抢注意力

现行 **SPEC-2.2a** 列表行仅定义「① 分类色圆点 ② 标题 ③ 相对时间」，并明文「severity…**不进列表行**」。故 `EventPanel.tsx` 对 M5.6 与 M0.35 渲染**完全同权**，用户无法从列表分辨轻重，违背 **D23 采纳的 grill §3 判据**「低价值条目不得与真正风险事件抢同等注意力」。

根因：SPEC-2.2a 把 severity **整体**排除出列表行——把「severity 的**数值/徽章展示**」（确属详情卡 M3）与「severity 的**轻重编码**」（应在 L0/列表即可感知，呼应 D23「多新/什么类」之外还需「多重」的注意力分配）混为一谈，一并逐出。

## 1. 设计选择 A —— 呈现门槛（入球 / 入流）

### 1.1 「入球」与「入流」统一为单一呈现集（论证，不默选）

任务措辞「入球/入流门槛」暗示两者可分。**本提案裁定二者统一**：一个事件是否被呈现是**单一准入判定**，其结果同时决定球面标记（§3）与事件流列表（SPEC-2.2/2.2a）是否出现该事件。理由：

1. **既有强契约 M2-13**（现 ✅ v0.2.5）明文「面板内渲染的列表条目数与当前球面已加载事件标记数**一一对应**」。拆分入球/入流门槛将直接破坏该 1:1。
2. BUG-023 的痛点**同时**是「球面 200+ 红点」**与**「事件流被淹没」——若只滤球不滤流，流仍被淹；只滤流不滤球，球仍拥挤。任何单侧门槛都不闭合缺陷。
3. D3「球为本体、列表从属」——列表是球的**从属镜像**，二者呈现同一事实集才自洽。

**被否**：入流门槛低于入球门槛（列表更全、球更净）。否决——重新引入「事件流被淹没」，且违背 D3 从属语义与 M2-13。

> 注：**聚合**（D23 L0）会在球面把密集簇合并为簇标记，届时「球面标记数」与「列表条目数」的对应关系需在**聚合提案**内另行定义；本提案（聚合前）维持 1:1，聚合的列表呈现留给该另案（§4）。

### 1.2 门槛在何处实现 —— USGS 换 M2.5+ feed（源侧），并给通则

**建议：USGS 改接 USGS 官方 `2.5_hour.geojson` / `2.5_day.geojson`（M2.5+ 显著性 feed），门槛在源侧兑现。** 三点理由：

1. **极简**：零新增过滤代码，门槛即 feed 边界；`severityFromMag` 与字段映射一字不改。
2. **性能（SPEC-3.10）**：`2.5_day` 载荷约为 `all_day` 的十分之一量级——直接服务首包/上屏预算，微震本就不该占带宽。
3. **门槛值非任意**：M≥2.5 = USGS **自身发布**的显著性 feed 分级（`significant` / `4.5` / `2.5` / `1.0` / `all` 之一），是行业公认「值得记录、部分可感」的边界，比自拟阈值更可辩护、更利 QA 锚定。

**被否方案**：保留 `all_hour`/`all_day` + 客户端震级过滤。否决——每 60s 下载 250+ 条再丢弃逾九成（相对 SPEC-3.10 是净退步），换来的唯一好处「任意阈值」并不需要（M≥2.5 已充分论证；若须调整，USGS 另有 `1.0`/`4.5` 现成分级）。

**实现自由度补强（非 spec，DEV 可选）**：可在 `normalizeFeature` 追加一行 `mag < 2.5 → 丢弃` 兜底，使门槛契约在**本侧**闭合、且让 QA 能以构造的 M2.0 要素直接断言「被滤除」（源侧 feed 本身无亚门槛样本，断言会退化为空真）。此为一处比较的廉价防御，属 DEV/rev 裁量，不写进 spec。

### 1.3 各源门槛值（现有四源 + 未来源通则）

| 源 | 门槛 | 实现 | 依据 |
| --- | --- | --- | --- |
| **USGS**（SPEC-5.1） | **M≥2.5** | 源侧 M2.5+ feed（+可选客户端 `mag≥2.5` 兜底） | BUG-023 实证微震淹没；M2.5 = USGS 官方显著性分级边界 |
| **EONET**（SPEC-5.2） | **无**（呈现全部） | — | `status=open` 自然事件、severity 默认 2、量小（fixture 26 条），无淹没证据 |
| **GDACS**（SPEC-5.3） | **无**（呈现全部） | — | 源侧已按 `alertlevel` 甄别、量小，无淹没证据（Green 若未来实证为噪声，再经 §7 加门槛） |
| **LL2**（SPEC-5.5） | **无**（呈现全部） | — | `upcoming?limit=10`，量极小 |
| **未来源通则** | 默认**无**门槛 | 源侧预筛优先，否则客户端谓词 | 仅当出现**实证噪声**（典型候选：GDELT 报道类高频，SPEC-5.4 M3 细化查询时评估）才经 §7 声明门槛 |

**通则一句话**：*每源默认呈现全部归一化事件；仅当有实证噪声时，经 §7 为该源声明一个显著性门槛，能源侧预筛则源侧、否则客户端谓词，亚门槛事件既不入球也不入流。*

### 1.4 门槛 ≠ 聚合 ≠ severity 编码（三机制分工，回应 BUG-025 联动要求）

BUG-025 要求「与 BUG-023 门槛方案联动、避免重复解决同一噪声」。三机制**作用于不同群体、不同轴、不重叠**：

| 机制 | 作用群体 | 轴 | 解决的问题 | 归属 |
| --- | --- | --- | --- | --- |
| **呈现门槛**（本提案 A） | 全部归一化事件 | 二值 in/out | 「亚门槛噪声是否**存在**」 | SPEC-5.0a/5.1 |
| **severity 轻重编码**（本提案 B） | **已准入**的呈现集 | 连续视觉轻重 | 「已呈现者间**注意力**分配」 | SPEC-2.2a / 3.7 |
| **密集区聚合**（D23 L0） | 已准入且地理密集者 | 空间合并 | 「合法事件的**几何拥挤**」 | 另案（§4） |

一条 M0.35 微震只被**门槛**处理（准入前即除，永不到达编码/聚合）；一条 M2.5 小震过门槛后由**编码**压为 severity 1（弱化）；加州残余的多条 M2.5+ 由**聚合**收敛。**同一条事件只在相应阶段被处理一次，无重复解决**。

## 2. 设计选择 B —— 列表行 severity 轻重编码

### 2.1 语义档位：3 档，直接映射 severity∈{1,2,3}

`GeoEvent.severity`（SPEC-6.1）本就是全源统一的「轻重」字段，且 marker 侧 SPEC-3.7 已按其分 **3 档**（尺寸/脉冲随级递增）。列表行沿用同一 3 档、与球面**同源同粒度**，避免另立标准：**severity 1 = 最轻（弱化、注意力最低）→ severity 3 = 最重（最突出）**，单调递增。

**被否**：列表塌缩为 2 档（「弱化 vs 常态」）。否决——与 SPEC-3.7 marker 3 档不一致，球↔列表两视图对同一事件呈现不同档数，破坏 D3 从属镜像一致性。

### 2.2 编码维度：仅非色相通道、单调、与 SPEC-3.7 marker 同原则；具体值归 D25

- **仅用非色相通道**（尺寸/字重/透明度/行密度等）。**色相保持「仅表分类」**（SPEC-3.7 分类色为唯一色语义）——severity **不得**以改变色点色相或引入新颜色来表达，否则复现 **BUG-022** 式「第三种色语义」污染，亦违反 `M2-globe.md` §2.3 已立的「高亮不得引入新颜色编码语义」底线。
- **单调**：所选通道上，视觉突出度随 severity 单调不减，severity 1 严格最弱、severity 3 严格最突出。
- **与球面同原则**：SPEC-3.7 marker 用「尺寸 + 脉冲幅度」（非色相）编码 severity；列表沿用「非色相 + 单调 + severity 高者更突出」的**同一原则**（通道可不同——DOM 二维 vs 三维标记）。
- **具体承载通道与其取值**（选哪几路通道、字重差多少、色点尺寸差多少、透明度档位）属**视觉造型**，**一律留白，待 D25 审美官方案定值**（D24/D25 路径：审美官出配色与层级 → arch 提案改条款 → rev → pin）。

### 2.3 「severity 不作显式字段」vs「环境式轻重编码」的区分（改写 SPEC-2.2a 关键点）

现行 SPEC-2.2a「severity…不进列表行」把两件事一起禁掉了，须拆分：

- **禁**：severity 的**数值 / 文字 / 徽章**作为**显式行元素**（如「severity 3」「⚠⚠⚠」徽章）——这仍归详情卡（SPEC-2.3，M3），列表行**不加第四个显式元素**。
- **准**：severity 以**行的整体视觉轻重环境式编码**（§2.2）——不新增可读元素，只调制既有三要素/整行的呈现强度。

即：列表行仍是**三个显式元素**，但其**呈现强度**由 severity 调制。

### 2.4 行为泄漏禁区自查（rev 专查项）

| 内容 | 是否对外可见行为 | 归属 |
| --- | --- | --- |
| USGS 门槛 = M≥2.5；EONET/GDACS/LL2 无门槛；亚门槛不入球/入流 | 是 | **进 spec**（§3.1/3.2 SPEC-5.0a/5.1 文本） |
| 列表行以视觉轻重编码 severity（3 档、单调、仅非色相、severity 数值不作显式元素） | 是 | **进 spec**（§3.3 SPEC-2.2a 文本） |
| 门槛在源侧 feed 还是客户端谓词实现、是否加客户端兜底 | 否（实现私有） | DP / DEV 自由度（§5.3） |
| 承载 severity 的**具体通道**与其**取值**（字重/尺寸/透明度数值） | 是（视觉造型） | **留白，待 D25 审美官**（spec 经审美官→arch→§7 二次入正文） |

本提案**不**在任何 design-prompt 里私定 spec 之外的对外行为；对外行为全部落 §3 的 spec 修订文本，视觉造型值显式留白给 D25。

## 3. SPEC 条文修改稿（现行 / 建议 / 理由；可直接替换）

### 3.1 新增 **SPEC-5.0a 呈现门槛**（§5 跨源统一约束，紧随 SPEC-5.0）

**现行**：无。

**建议新文**：

> - **SPEC-5.0a 呈现门槛（surfacing floor，跨源统一约束）**：每源可声明一个**显著性门槛**；仅达到/高于门槛的归一化事件进入**呈现集**——同时决定球面标记（§3）与事件流列表（SPEC-2.2/2.2a）是否出现该事件，**球面与列表共用同一呈现集**（维持 SPEC-2.2「事件流」从属语义与标记↔列表一一对应）。低于门槛的事件**既不入球也不入流**。门槛以最省的可用手段实现——源侧提供预筛（feed/查询参数）时优先源侧（见 SPEC-5.1），否则客户端谓词过滤；**实现位置属实现自由度**，对外契约仅为「亚门槛事件不呈现」。**无噪声问题的源不声明门槛（呈现全部事件）**：EONET/GDACS/LL2 现均无门槛。新增或变更任一源的门槛属 spec 变更（§7），须以**实证噪声**为据。呈现门槛只做**准入**（二值 in/out），与密集区**聚合**（D23 L0）、列表行 severity **轻重编码**（SPEC-2.2a）分属不同机制、互不替代。

**理由**：把「哪些归一化事件才被呈现」这一原本空白的跨源契约显式化（BUG-023 根因）；统一入球/入流为单一呈现集（§1.1，锚定 M2-13）；给未来源可扩展的通则（§1.3）；显式与聚合/编码划界（§1.4）避免重复解决。

**建议新文**：

> - **SPEC-2.2a** 事件流面板列表行：SPEC-2.2 事件流面板的每个列表行显示三项——① 分类色圆点（取 SPEC-3.7 对应 category 色）；② 事件标题（SPEC-6.1 `title`）；③ 相对时间（由 SPEC-6.1 `ts` 相对当前时刻派生，如「3 分钟前」，格式细节属实现自由度）。**行的整体视觉轻重编码事件 severity（SPEC-6.1，severity∈{1,2,3}）**：跨三档**单调递增**——severity 1 最轻（弱化、注意力最低）、severity 3 最重（最突出）；编码**仅用非色相通道**（尺寸/字重/透明度/行密度等），**色相仅表分类（SPEC-3.7），severity 不引入任何新增色语义**（与 marker 侧 SPEC-3.7「尺寸/脉冲随级递增」同原则，避免 BUG-022 式第二色语义）；**具体承载通道与其取值由 D25 审美官方案定（待 D25 定值）**。列表按 `ts` 倒序（最新在上，与「事件流」语义一致；`ts` 仅供展示/排序，SPEC-6.3①）。无事件时显示空状态文案「暂无事件」。severity 的**数值/文字/徽章不作为显式行元素**（其数值展示属详情卡 SPEC-2.3，M3）——行内仅以上述视觉轻重环境式编码 severity 档位，不新增可读元素；地点、摘要、信源等完整字段属详情卡（SPEC-2.3，M3），不进列表行。列表所呈现的事件集受 SPEC-5.0a 呈现门槛约束。

**理由**：拆开「severity 数值展示（禁，归详情卡）」与「severity 轻重编码（准，环境式）」（§2.3）；3 档、单调、非色相、与 SPEC-3.7 同原则（§2.1/2.2）满足 D23 grill §3；视觉造型值显式留白 D25（§2.4）；补锚 SPEC-5.0a 呈现门槛。

## 4. 极简边界（本提案**不做**）

- **不含密集区聚合**（D23 L0 强制项）：属另案；聚合落地后「球面簇标记 ↔ 列表条目」的对应关系在该案定义，本提案维持聚合前 1:1（§1.1）。

- **不含 marker 白斑**（BUG-022，AdditiveBlending 饱和）：另案；本提案的「非色相/唯一色语义」约束与其同向，但不代解。

- **不给 EONET/GDACS/LL2 加门槛**（无实证噪声，§1.3）；不预建门槛「可配」参数化（未被需要）。
- **不定任何视觉造型值**（字重/尺寸/透明度/色点大小等）——留 D25。

## 7. 遗留风险 / 待仲裁项

1. **USGS 端点变更是否接受**（`all_*` → `2.5_*`，§1.2/3.2）：本提案建议源侧换 feed；rev 若倾向保留 `all_*` + 客户端过滤，则 §3.2 端点不改、改为在 SPEC-5.1 增「客户端 M≥2.5 过滤」子句（性能次优，见 §1.2 被否理由）。待 rev 拍板。
2. **门槛值 M≥2.5 vs M≥3.0**：本提案取 2.5（USGS 官方分级、保留有感小震）；rev 若判「更少更准」可上调至 3.0（USGS 无 3.0 现成 feed，须客户端过滤或取 4.5 feed——4.5 会丢弃全部 severity 1 地震，过激）。待 rev 拍板。
3. **GDACS Green 是否需门槛**：本提案暂不设（无实证淹没，§1.3）；若 rev/后续实测判 Green 为噪声，另经 §7 加门槛，不在本提案。
4. **列表 severity 编码依赖 D25 定值**：SPEC-2.2a 语义（3 档/单调/非色相/数值不作显式元素）本提案即可 pin；**具体通道+取值待 D25 审美官**——故新增列表 severity 场景的**不变量 B**（单调）需待 D25 落地方可完整断言，A/C 现即可断言。建议此视觉实现随 D24/D25 视觉批次统一落地，不单独抢跑。
5. **依赖裁决**：本提案须经 rev 仲裁通过后，由 orch 应用 SPEC-5.0a/5.1/2.2a 正文 + §0 修改记录 + `make pin-spec` + 同步 testplan/FM/DP，dev 方可改 `usgs.ts` 端点与 `EventPanel.tsx` 编码。在此之前 dev 不得抢改（现 `all_*` feed 与无 severity 编码保持，M2-05/M2-13 现 ✅ 暂不动）。
