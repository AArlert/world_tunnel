## 1. 新版里程碑划分

M0（基建）、M1（地球仪）**已完成并 tag**，不重排。以下为 M2 起重排结果，v1.0 = 阶段一完整发布（零服务器 + 风格包收入）。

### M2 — 事件数据核心与首屏（T1 信源 · 矢量默认 · 基础个性化 · 缓存启动）
**范围**
- 数据核心：GeoEvent 归一化模型、多源 scheduler（独立轮询/限流/退避）、去重与过期、本地缓存（IndexedDB）。
- T1 自带坐标信源（D13）：USGS / EONET / GDACS / **LL2（由 M3 前移至 M2**，属 T1 自带坐标）。
- 球面标记层：分类色表 + severity 分级 + 脉冲环，instancing/点精灵；**回补 SPEC-3.4「大气不遮挡真标记」（K-2/R-1，非材质代理断言）与 SPEC-3.8「≥200 标记 instancing/帧率」（R-2）**。
- 右侧事件流面板（SPEC-2.2 主从）；**列表 hover/选中 ↔ 球面标记高亮双向联动（SPEC-7.4 列表联动分片，K-1/R-3；点击飞行 + 详情卡属 M3）**。
- **基础分类过滤**（六 category 开关，D6）——「选择即定位」的最小形态。
- **缓存优先启动 + 呼吸式过渡**（D9）。
- **性能预算基线**（D10）：矢量默认落地以兑现首包 ≤2MB。
- **默认矢量风格**（D12）：替换 M1 的卫星默认为矢量昼夜/晨昏线；卫星纹理降级为可选（进 M4 天气付费包）。
- 顶栏品牌名 **Worlens** + UTC 时钟（D4 + BUG-014）。
- **合规调研启动**：对 M2 接入源（USGS/EONET/GDACS/LL2）出条款核查记录。
**完成判据要点**：SPEC-5.0/5.1/5.2/5.3/5.5、SPEC-6.1/6.2/6.3、SPEC-3.4（不遮挡真标记回补·K-2）、SPEC-3.7/3.8（≥200 标记 instancing/帧率·R-2）、SPEC-2.1/2.2、SPEC-2.4（基础分类）、SPEC-3.2（矢量重写 + 矢量默认昼夜·A3）、SPEC-3.10（基线）、SPEC-3.11（缓存启动）、SPEC-7.4（列表↔标记联动分片·K-1）、SPEC-8.1（分类子集）对应 testplan 场景全 ✅ + `make regress` 100% + signoff-M2。

### 2.3a SPEC-2.5 搜索（**新增·K-3**）
- 背景：现行 spec 仅在 §9 负向框定「全文检索非目标（搜索仅限当前缓存事件与地名）」，界定了搜索**边界**却无任一正文条目肯定「存在搜索」并定义其对外可见形态。搜索作为对外可见功能仅靠非目标条款括注存在，属行为泄漏禁区边缘情形（REV-005 K-3）。本条补定义性正文，使搜索有正向 spec 依据；边界仍以 §9 为准。
- 建议文本（新增 SPEC-2.5，M3）：
  - **入口**：事件流面板（SPEC-2.2）顶部搜索框。
  - **范围**：仅检索**当前本地缓存事件**（标题/信源名/已解析地名字段）与**本地地名 gazetteer**（SPEC-5.8 T2 同一份精简 GeoNames）；不发起网络请求、不做全文检索（与 §9 一致）。
  - **命中行为**：命中事件 → 定位到球面对应标记并可打开详情卡（SPEC-2.3，飞行走 SPEC-7.4）；命中地名 → 相机飞至该坐标（SPEC-7.4）。
  - **空状态**：无匹配时给明确空状态提示（如「当前缓存内无匹配」）。
- 依据：D1（glanceable 快速定位）、§9（搜索边界）。（搜索交互与范围为对外可见行为，入 spec 正文；本条经 rev 仲裁应用后，K-3 的「M3 前闭合」义务即闭合。）
- 备选（若产品决定阶段一不做搜索）：从 FM-14 与 §1 M3 范围移除「搜索」，理由记于此，§9 括注保留为未来预留——**本提案取「新增 SPEC-2.5」而非移除**，因 D1 的快速定位价值与 §9 既有边界均指向保留搜索。

### 2.4 SPEC-3.1 初始视角（改·追加）
- 现行：初始视角 = (lat 0, lon 0) 正上方、距 3.2、地球初始自转角 0。
- 建议：追加「**开屏锚定**：若用户已通过首启引导（SPEC-8.6）或 watchlist 选定关注区域，则冷启动初始视角改为近景该区域（覆盖本条默认几内亚湾视角）；未选定时沿用默认。」
- 依据：D8。

### 2.5 SPEC-3.2 昼夜混合与风格系统（**重写**）
- 现行：昼夜混合基于卫星 `earth_day.jpg`/`earth_night.jpg` 自定义 shader，晨昏线软过渡带 t∈[-0.1,+0.1]。默认即卫星纹理。
- 建议：将「昼夜/晨昏线」与「地表底图风格」解耦——
  ①**昼夜/晨昏线为跨风格能力**：任一风格下 `t=dot(N,sunDir)`、软过渡带 t∈[-0.1,+0.1]（半宽约 5.7°）smoothstep 混合、纹理/资源就绪前深色占位（保留现行加载与失败表现）。
  ②**默认风格 = 轻量矢量**（科技感、轻呼吸、静谧）：以矢量海岸线/网格 + 昼夜明暗表达晨昏线，免大纹理。
  ③**卫星昼夜底图退出默认**，并入付费天气风格包（SPEC-3.9）。
- 依据：D12、D3（矢量静谧兑现宁静基调）。
- **影响**：testplan M1-05/M1-06/M1-14 术语（昼纹理/夜纹理）需复核归属——见第 4 节。

### 2.6 SPEC-3.3 夜景灯光（改）
- 现行：夜半球显示夜纹理城市灯光（增益 ≥1.5）；昼半球不显示。
- 建议：追加「城市灯光为**卫星/天气风格包专属**（依赖夜纹理）；矢量默认风格以矢量点/辉光表达夜面，不依赖夜纹理。『昼半球不叠加灯光、昼夜为混合而非叠加』的约束跨风格保持。」
- 依据：D12。

### 2.7 SPEC-3.9 风格与风格包（**新增**）
- 建议文本：
  - 默认风格 = 轻量矢量（免大纹理，兑现首包预算 SPEC-3.10）。
  - 付费风格包 = **天气云图版**（卫星底图 + 定期从免费源更新的云图，数据源待调研并写入 ASSETS.md）、**像素**、**艺术**。
  - **定价：¥6 / $1.99 一口价，永久解锁全部付费风格（天气/像素/艺术）**；解锁状态本地持久化。
  - 风格包资源**懒加载**：未解锁/未选用不下载，不计入首包。
  - 风格切换入口在设置面板（SPEC-2.4）。
- 依据：D12。（价格与解锁范围为对外可见行为，故入 spec 正文。）

### 2.8 SPEC-3.10 性能预算（**新增**；与 SPEC-3.8 并列）
- 建议文本（数字定稿）：
  - 中端手机冷启动 → 可交互地球 **≤3s**；
  - 缓存事件上屏 **≤1s**（SPEC-3.11 缓存优先）；
  - 首包 **≤2MB**（不含可选纹理/风格包）；
  - 风格包与卫星/天气纹理一律懒加载，不计入首包。
  - 验收基准环境与量测方法由 M5 testplan 场景定义。
- 依据：D10。

### 2.9 SPEC-3.11 缓存优先启动与呼吸式过渡（**新增**）
- 建议文本：
