## 1. 产品概述

个人专属新闻信息流地球仪（对标 Ambient News Globe 形态）：可交互 3D 地球实时呈现全球事件——
突发新闻、自然灾害、地区冲突与人道危机，外加火箭发射、航班动态、加密行情。
点击事件可见附带信源链接的精简摘要；watchlist/分类过滤只看关心的。
产品对外名 **Worlens**，中文 slogan「看见世界正在发生」（仓库内部代号 world_tunnel 保留）；
产品定位为 glanceable「世界的表盘」：三秒看见此刻地球上正在发生什么，且只看用户关心的部分。
单用户、纯客户端直连免费 API；Web 优先（Vite+React+three.js），经 Capacitor 打 iOS 包（M6）。
阶段一零服务器，**唯一例外 = 匿名遥测心跳（SPEC-8.7）**；商业模式 = 风格包一次性解锁（SPEC-3.9）。
任何产品功能上服务器须过阶段二触发条件（净收入月均 ≥ 预估服务器月开销 ×50%），届时另起 spec 提案。

## 2. 界面布局

- **SPEC-2.1** 顶栏（48px）：品牌名「**Worlens**」（M2）· 加密行情 ticker（M3，SPEC-5.7）· **UTC 时钟（M2，实时刷新，格式 HH:MM:SS UTC）**。
- **SPEC-2.2** 主区：全屏地球 canvas；右侧悬浮事件流面板（宽 300px，可折叠）。
- **SPEC-2.3** 事件详情卡：点击球面标记或列表条目弹出浮层——标题、分类徽章、severity、时间（相对+绝对）、地点、摘要、信源链接列表（≥1 条）。
- **SPEC-2.4** 过滤与设置：①**基础分类过滤**（六 category 开关）前移至 M2，作为首屏可见控件（不进设置面板）；②设置入口（M4）：watchlist 管理、高级过滤、图层开关（航班）、**风格包切换（SPEC-3.9）**；③设置面板**不含**「摘要模式（信源/Claude）」——AI 摘要后置为付费能力（§9、SPEC-8.5）。
- **SPEC-2.5** 搜索（M3）：入口为事件流面板（SPEC-2.2）顶部搜索框。范围仅限**当前本地缓存事件**（标题/信源名/已解析地名字段）与**本地地名 gazetteer**（与 SPEC-5.8 T2 同一份精简 GeoNames）；不发起网络请求、不做全文检索（与 §9 一致）。命中事件 → 定位到球面对应标记并可打开详情卡（SPEC-2.3，飞行走 SPEC-7.4）；命中地名 → 相机飞至该坐标（SPEC-7.4）。无匹配时给明确空状态提示（如「当前缓存内无匹配」）。

## 3. 地球视觉规格

- **SPEC-3.1** 几何：球半径 1.0，SphereGeometry 分段 ≥64；相机 fov 45°，初始距离 3.2。
  初始视角：相机位于 (lat 0, lon 0) 正上方、距球心 3.2、视线指向球心，上方向为 +Y（即初始画面正中为几内亚湾方向的半球中心）；地球本体初始自转角为 0。
  **开屏锚定（M3）**：若用户已通过首启引导（SPEC-8.6）或 watchlist 选定关注区域，则冷启动初始视角改为近景该区域（覆盖本条默认几内亚湾视角）；未选定时沿用默认。
- **SPEC-3.2** 昼夜混合与风格系统：「昼夜/晨昏线」与「地表底图风格」解耦——
  ①**昼夜/晨昏线为跨风格能力**：任一风格下自定义 shader `t = dot(N, sunDir)`，晨昏线软过渡带 t ∈ [-0.1, +0.1]（半宽约 5.7°）内 smoothstep 混合。
  纹理/资源就绪前：球体以深色纯色（近夜面色）渲染，**不显示**加载文案、进度指示或骨架动画；资源加载完成后直接替换，无淡入过渡。加载失败不重试、不弹错误 UI，保持占位渲染（错误细节仅进 console）。
  ②**默认风格 = 轻量矢量**（科技感、轻呼吸、静谧）：以矢量海岸线/网格 + 昼夜明暗表达晨昏线，免大纹理。
  ③**卫星昼夜底图退出默认**：昼纹理 `earth_day.jpg`、夜纹理 `earth_night.jpg`（等距圆柱投影，来源见 public/assets/ASSETS.md）并入付费天气风格包（SPEC-3.9）。
- **SPEC-3.3** 夜景灯光：夜半球显示夜纹理城市灯光（亮度增益 ≥1.5）；昼半球不显示灯光。
  城市灯光为**卫星/天气风格包专属**（依赖夜纹理）；矢量默认风格以矢量点/辉光表达夜面，不依赖夜纹理。「昼半球不叠加灯光、昼夜为混合而非叠加」的约束跨风格保持。
- **SPEC-3.4** 大气：菲涅尔边缘辉光，主色 `#4a90d9`，从球缘向外衰减；不遮挡标记。
- **SPEC-3.5** 星空：程序化点星 ≥1500 颗，分布于半径 ≥40 球壳，随相机旋转（不随地球自转）。
- **SPEC-3.6** 纹理经度对齐：格林尼治（lat 0, lon 0）的球面标记必须落在昼纹理的非洲西侧几内亚湾位置（M1 校准场景验证）。
- **SPEC-3.7** 事件标记分类色表与分级：

| category | 颜色 | 含义 |
| --- | --- | --- |
| disaster | `#ff4d4f` | 自然灾害（地震/风暴/野火/火山…） |
| conflict | `#ff7a45` | 地区冲突 |
| humanitarian | `#ffc53d` | 人道危机 |
| news | `#40a9ff` | 突发新闻 |
| launch | `#b37feb` | 火箭发射 |
| flight | `#5cdbd3` | 航班（图层） |

  severity ∈ {1,2,3}：标记基础尺寸与脉冲光环幅度随级别递增；severity 3 必须有持续脉冲环。

- **SPEC-3.8** 性能：桌面 Chrome 目标 60fps；标记 ≥200 个时用 instancing/点精灵，不逐事件建 Mesh。
- **SPEC-3.9** 风格与风格包：默认风格 = 轻量矢量（免大纹理，兑现首包预算 SPEC-3.10）。付费风格包 = **天气云图版**（卫星底图 + 定期从免费源更新的气象云图叠加，数据源待调研并写入 ASSETS.md）、**像素**、**艺术**。**定价：¥6 / $1.99 一口价，永久解锁全部付费风格（天气/像素/艺术）**；解锁状态本地持久化（SPEC-8.8）。风格包资源**懒加载**：未解锁/未选用不下载，不计入首包。风格切换入口在设置面板（SPEC-2.4）。
- **SPEC-3.10** 性能预算（与 SPEC-3.8 并列）：中端手机冷启动 → 可交互地球 **≤3s**；缓存事件上屏 **≤1s**（SPEC-3.11）；首包 **≤2MB**（不含可选纹理/风格包）；风格包与卫星/天气纹理一律懒加载，不计入首包。验收基准环境与量测方法由 M5 testplan 场景定义（M2 建基线量测）。
- **SPEC-3.11** 缓存优先启动与呼吸式过渡：冷启动即渲染上次本地缓存事件（球面标记），不空屏等待网络。首次刷新完成后以**呼吸式过渡**收敛：已过期/被替换的旧标记渐隐熄灭、最新事件渐亮，不做整屏重绘闪烁。缓存从「可重建、不承诺」升格为**启动路径的一部分**（SPEC-8.4）。

## 4. 天文计算（晨昏线）

- **SPEC-4.1** 太阳直下点纬度 = 太阳赤纬 δ（Cooper 公式）：`δ = 23.44° · sin(2π·(284+N)/365)`，N 为年积日。精度要求 ±1°（视觉用途）。
  N 为年积日，按 **UTC 日期**计，1 月 1 日 N=1；闰年 2 月 29 日照常顺次计入（公式分母恒为 365，不因闰年调整）。SPEC-4.2 的 N 取同一定义。
- **SPEC-4.2** 均时差（分钟）：`EoT = 9.87·sin(2B) − 7.53·cos(B) − 1.5·sin(B)`，`B = 2π·(N−81)/364`。
- **SPEC-4.3** 太阳直下点经度 = `−15° · (h_UTC − 12 + EoT/60)`，h_UTC 为 UTC 小数小时；结果归一化到 (−180°, 180°]。
- **SPEC-4.4** 单测锚点（容差 ±1°）：春分 3-20 δ≈0；夏至 6-21 δ≈+23.44°；冬至 12-21 δ≈−23.44°；均时差全年幅度 |EoT| ≤ 17 分钟；EoT≈0 的日期（如 4-15 前后）UTC 正午直下点经度 ≈ 0°。
- **SPEC-4.5** sunDir 向量 = 直下点 (lat, lon) 经 SPEC-6.2 坐标约定转成的单位向量；shader 每帧按当前时刻更新（可降频至 1 次/分钟）。

## 5. 数据源契约（全部免费、无需 key）

统一约束：**SPEC-5.0** 每源独立轮询与限流预算；HTTP 失败指数退避（基础间隔×2^n，上限 30min）；支持 ETag/Last-Modified 的源带条件请求；任何源故障不得影响其他源与渲染。

- **SPEC-5.1 USGS 地震** → category `disaster`
  - `https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_hour.geojson`（启动时先拉 `all_day.geojson` 回填）
  - 轮询 60s。映射：`id=usgs:{feature.id}`；title=`M{mag} {place}`；lat/lon=geometry；ts=properties.time；url=properties.url。
  - severity：mag<4.5→1，4.5≤mag<6→2，mag≥6→3。
- **SPEC-5.2 NASA EONET 自然事件** → category `disaster`
  - `https://eonet.gsfc.nasa.gov/api/v3/events?status=open&days=7`，轮询 300s。
  - 映射：`id=eonet:{event.id}`；坐标取 geometry 数组中**时间最新**的一条——其 `type` 为 `Point` 时取该点 `[lon, lat]`；为 `Polygon`/`MultiPolygon` 时取该 geometry **全部坐标点的经纬度包围盒中心** `((minLon+maxLon)/2, (minLat+maxLat)/2)` 作为 (lat, lon)（`Point` 为其单点退化情形）。该降维为可视化落点近似、不追求面积质心；跨 ±180° 经线的多边形（EONET 极罕见）落点可能偏移，属已知限界，如需可后续另行提案精化。categories[0].title 进 summary；sources[].url 为信源。severity 默认 2。
- **SPEC-5.3 GDACS 灾害/人道** → category `disaster` 或 `humanitarian`（事件类型含 DR/FL 且带人道响应字段时）
  - `https://www.gdacs.org/gdacsapi/api/events/geteventlist/MAP`，轮询 300s。
  - alertlevel Green/Orange/Red → severity 1/2/3。`id=gdacs:{eventid}`。
- **SPEC-5.4 GDELT 突发新闻/冲突** → category `news` 或 `conflict`（M3 起点细化查询词与判类规则，此处 pin 接口形态）
  - `https://api.gdeltproject.org/api/v2/doc/doc?query=...&mode=ArtList&format=json&maxrecords=50`，轮询 180s。
  - 定位分层（SPEC-5.8）：①优先采用 GDELT 自带地理编码结果（T3，不自研 NER）；②仍无坐标时用本地 GeoNames gazetteer（T2）按国家/城市兜底；③再无定位则丢弃（SPEC-5.4a）。
- **SPEC-5.5 Launch Library 2 火箭发射** → category `launch`（M2：属 T1 自带坐标源，随首批 provider 接入）
  - `https://ll.thespacedevs.com/2.2.0/launch/upcoming/?limit=10&mode=list`，轮询 1800s（免费额 15 req/h，预算 ≤2 req/h）。
  - 坐标取发射工位；T-24h 内 severity 2，T-1h 内 3，其余 1。`id=ll2:{launch.id}`。
- **SPEC-5.6 OpenSky 航班图层** → category `flight`（默认关闭，开启才轮询）
  - `https://opensky-network.org/api/states/all?lamin=…&lomin=…&lamax=…&lomax=…`（当前视口 bbox），轮询 60s，匿名额度内；关图层立即停拉。severity 恒 1。
- **SPEC-5.7 CoinGecko 行情 ticker**（非地理事件，只进顶栏）
  - `https://api.coingecko.com/api/v3/simple/price?ids=bitcoin,ethereum&vs_currencies=usd&include_24hr_change=true`，轮询 60s；币种清单 M4 起可配置。
- **SPEC-5.8 信源解析分层 T1–T4**（能力越高越晚做）：
  - **T1** 自带坐标源（USGS/EONET/GDACS/LL2）——M2。
  - **T2** 结构化地名 → 本地精简 GeoNames gazetteer 查表——M3。
  - **T2.5** 用户自定义关键词→地点规则（如「Google→硅谷」），纯本地——M3。
  - **T3** 自由文本地理解析不自研，直接采用 GDELT 地理编码结果——M3。
  - **T4** 实体→地点智能解析**后置为付费能力**；早期以**人肉钉图**替代（用户订阅 feed 时自选钉定地点）——后置。
- **SPEC-5.9 自定义 RSS 信源**：归**原生端（Capacitor）能力**——原生请求不受 CORS 限制，支持任意 feed（M6）。web/PWA 端仅支持 **CORS 开放的 feed**，对不支持者**明确提示**其受浏览器同源策略限制（M5）。内容展示**以 feed 自带内容为限**（feed 含全文则可展示全文）；尊重 feed 声明的转载限制；**不抓取 feed 之外的正文**。自定义 feed 事件缺坐标时走人肉钉图（SPEC-5.8 T4 替代）。

## 6. GeoEvent 模型与坐标约定

- **SPEC-6.1** 归一化模型（全源统一）：

```ts
interface GeoEvent {
  id: string          // "{source}:{原始id}"，全局唯一，跨轮询去重键
  category: 'disaster' | 'conflict' | 'humanitarian' | 'news' | 'launch' | 'flight'
  severity: 1 | 2 | 3
  title: string
  summary: string     // 信源自带描述；可为空串
  urls: string[]      // 信源链接，详情卡展示，≥1（flight 可为空）
  lat: number         // WGS84 度
  lon: number
  ts: number          // epoch ms（事件时间；无则用抓取时间）
  source: string      // 'usgs' | 'eonet' | 'gdacs' | 'gdelt' | 'll2' | 'opensky'
}
```

- **SPEC-6.2** 球面坐标约定（three.js 右手系，y 上）：北极 (90,·)→+Y；(0,0)→+Z；(0,90°E)→+X。实现：`src/globe/geo.ts` latLonToVector3。
- **SPEC-6.3** 同 id 事件再次出现视为更新（覆盖 ts/severity/summary），不新增标记。过期与保留：①默认过期窗 **48–72h** 无更新移除（具体值视本地存储预算定，可配）；flight 60s 不变。②**用户收藏的事件永久本地保存**，不受过期窗影响（M4）。③为未来时间滑块**预留缓存窗口**（滑块 UI 不进早期版本，见 §9）。

## 7. 交互规格

- **SPEC-7.1** 拖拽旋转：水平绕 Y 无限制；垂直视角限制在纬度 ±85°；释放后惯性衰减（阻尼系数 ≈0.95/帧，**以 60fps 为基准**）。
  拖拽与其后的惯性作用于**相机**（绕球心的方位角/仰角），地球本体不因拖拽转动。
- **SPEC-7.2** 缩放：滚轮/双指，相机距离 ∈ [1.8, 6]。
- **SPEC-7.3** 空闲自转：无输入 10s 后缓慢自转（≈0.02°/帧，**以 60fps 为基准**）；任何输入立即停。
  空闲自转作用于**地球本体**（绕地轴，即 SPEC-6.2 的 +Y 轴），相机与星空不动；自转期间晨昏线与地理位置的相对关系保持不变（晨昏线由 SPEC-4.5 的真实时刻驱动，不随自转移动）。
- **SPEC-7.4** 点击标记/列表条目：相机 800ms 缓动飞行到该事件上空 + 弹详情卡（SPEC-2.3）；列表 hover/选中与球面标记高亮双向联动。
- **SPEC-7.5** 时间基准：SPEC-7.1 / 7.3 中以「每帧」表述的常量均以 60fps（帧间隔 1/60 s）为基准；实际帧率不同时，单位**时间**内的衰减比例与自转角速度保持一致，不随设备刷新率变化。

## 8. 个性化、引导与通知

- **SPEC-8.1** watchlist 三类条目：地点（圆域：中心+半径 km）、主题（关键词，匹配 title+summary）、分类（category 集合）。多条目为「或」关系。
  **分类（category 集合）过滤前移至 M2** 作为首屏基础控件；地点（圆域）与主题（关键词）仍为 M4 高级 watchlist。
- **SPEC-8.2** 过滤模式：全部事件 / 仅 watchlist 命中。命中事件在列表与球面均有视觉强调。
- **SPEC-8.3** 通知：新事件命中 watchlist → Web Notification（M5 换 Capacitor Local Notifications）。**语义为「应用打开/前台时补齐」**，不承诺后台推送。
- **SPEC-8.4** 持久化：watchlist/设置存 IndexedDB；事件缓存**升格为启动路径的一部分**（SPEC-3.11）；用户收藏事件永久保存（SPEC-6.3）。仍不承诺离线数据完整性。
- **SPEC-8.5** AI 摘要（后置）：AI 摘要/聚类/区域周报等消化能力**后置为付费功能（阶段二前后）**，前提是数据层（融合/去重/清洗）先做扎实；早期版本不含，设置面板不出摘要模式开关（SPEC-2.4③、§9）。
- **SPEC-8.6** 首启引导（M3）：首次打开展示「你想看什么」引导，采集**两个并列维度**——
  ①**垂类分流**（冲突观察/航天/灾害等主题类别）：驱动默认分类过滤（SPEC-8.1）；
  ②**粗粒度地理关注**（关注地/大区域，如「东亚」「中东」「北美」或一处关注城市的粗粒度圆域）：驱动开屏锚定（SPEC-3.1）。此维度为 M3 开屏锚定的**唯一数据源**；M4 的地点圆域 watchlist（SPEC-8.1）在此粗粒度关注上**再精化**。
  引导按核心界面前三重要设计；两维度选择均本地持久化、可在设置中修改。可跳过；跳过或未选地理关注时，开屏锚定回落 SPEC-3.1 默认视角（几内亚湾）。
- **SPEC-8.7** 匿名遥测心跳（M5）：每用户一个**随机码**，除心跳 ping 外**不追踪任何个人信息**，无账号、无设备指纹。**明示为阶段一唯一服务端组件，不承载任何产品功能**；任何功能上服务器须过阶段二触发条件（SPEC-1）。自订低配服务器；发布前实现；**用户可关闭**。首次运行须在 UI 明示遥测存在与关闭入口。
- **SPEC-8.8** 支付与解锁：**iOS** 走 App Store 内购解锁风格包（M6）；**其余平台（含 PWA/桌面）**走 Gumroad / Lemon Squeezy 类代销平台发**许可码**，**本地校验、跨设备同码、无账号系统**（M4）。解锁状态本地持久化；解锁范围见 SPEC-3.9（一口价永久解锁全部付费风格）。
