## 1. 产品概述

个人专属的世界表（instrument-object「仪物」）——一件交付安心、而非交付信息条目的安静仪器。可交互 3D 地球实时呈现全球事件——
突发新闻、自然灾害、地区冲突与人道危机，外加火箭发射、航班动态。
点击事件可见附带信源链接的精简摘要；守望/分类过滤只看关心的。
产品对外名 **Worlens**，中文 slogan「看见世界正在发生」（仓库内部代号 world_tunnel 保留）；
产品定位为 glanceable「世界的表盘」：三秒看见此刻地球上正在发生什么，且只看用户关心的部分。
单用户、纯客户端直连免费 API；Web 优先（Vite+React+three.js），经 Capacitor 打 iOS 包（M6）。
阶段一零服务器，**唯一例外 = 匿名遥测心跳（SPEC-8.7）**；商业模式 = 风格包一次性解锁（SPEC-3.9）。
任何产品功能上服务器须过阶段二触发条件（净收入月均 ≥ 预估服务器月开销 ×50%），届时另起 spec 提案。

## 2. 界面布局

- **SPEC-2.1** 顶栏（48px）：品牌名「**Worlens**」（M2）· **UTC 时钟（M2，实时刷新，格式 HH:MM:SS UTC）**。
- **SPEC-2.2** 主区与表盘默认态：主区为全屏地球 canvas，占满视口宽高。
  **表盘默认态（冷启动静息态）以地球为视觉主体，静息核心 = 地球 canvas + 事件光柱标记（SPEC-3.7）+ 顶栏 UTC 时钟（SPEC-2.1「时间」）**；
  **事件流面板不常驻画面，退位为默认收起的「今日刻痕」抽屉**（「球为本体、列表从属」D3 走到底：静息即球、时间、光柱，静而克制）。
  抽屉冷启动即收起态，提供展开/收起双向切换控件；展开后呈现事件流列表（SPEC-2.2a 列表行），
  沿用现有事件流面板形态——展开态悬浮于 canvas 之上、不挤占 canvas，展开宽度约 300px。
  抽屉展开态底亮度不得高于夜半球底面（SPEC-3.2a 夜端 `#0d1827`），使抽屉退后、球为本体（D3）。
  抽屉对外名「今日刻痕」（product-vision 基准，最终措辞属 aes 制定项）；抽屉展开态具体底色/边框/圆角/质感（实底或玻璃）、
  收起态触发控件的形态与位置、展开/收起动效属 UI 实现自由度，其中视觉呈现取值交 aes 制定（后续 /aes-review 批次）。
  **本条重定义仅及事件流面板（→今日刻痕抽屉）与表盘默认态**；顶栏（SPEC-2.1）与基础分类过滤六开关（SPEC-2.4① 首屏可见控件）的首屏呈现不由本条改动。

## 3. 地球视觉规格

- **SPEC-3.1** 几何：球半径 1.0，SphereGeometry 分段 ≥64；相机 fov 45°，初始距离 3.2。
  初始视角：相机位于 (lat 0, lon 0) 正上方、距球心 3.2、视线指向球心，上方向为 +Y（即初始画面正中为几内亚湾方向的半球中心）；地球本体初始自转角为 0。
  **开屏锚定（M3）**：若用户已通过首启引导（SPEC-8.6）或守望选定关注区域，则冷启动初始视角改为近景该区域（覆盖本条默认几内亚湾视角）；未选定时沿用默认。
- **SPEC-3.2** 昼夜混合与风格系统：「昼夜/晨昏线」与「地表底图风格」解耦——
  ①**昼夜/晨昏线为跨风格能力**：任一风格下自定义 shader `t = dot(N, sunDir)`，晨昏线软过渡带 t ∈ [-0.1, +0.1]（半宽约 5.7°）内 smoothstep 混合。**过渡带只约束晨昏线锐利度（品质细节）；昼半球底面亮度在过渡带之外仍随离日角连续柔和衰减——次日点（t=1）最亮、向晨昏线方向渐暗，使昼半球呈「被定向光照亮的球面」而非平涂圆盘。衰减曲线形态（Lambert/余弦式等）属实现自由度；半球级明暗对比为目标（D24），晨昏线锐利与否非关键。**
  纹理/资源就绪前：球体以深色纯色（近夜面色）渲染，**不显示**加载文案、进度指示或骨架动画；资源加载完成后直接替换，无淡入过渡。加载失败不重试、不弹错误 UI，保持占位渲染（错误细节仅进 console）。
  ②**默认风格 = 轻量矢量**（科技感、轻呼吸、静谧）：以矢量海岸线/网格 + 昼夜明暗表达晨昏线，免大纹理。
  ③**卫星昼夜底图退出默认**：昼纹理 `earth_day.jpg`、夜纹理 `earth_night.jpg`（等距圆柱投影，来源见 public/assets/ASSETS.md）并入付费天气风格包（SPEC-3.9）。
- **SPEC-3.2a** 矢量默认风格视觉参数：矢量默认风格（SPEC-3.2②/3.3）的对外可见取值如下；昼夜/晨昏线仍照 SPEC-3.2①（t/k、过渡带 t∈[-0.1,+0.1]、非叠加 SPEC-3.3）在下列两端间插值，本条 pin 被混合两端的取值、网格密度（默认隐藏见下）与昼夜半球对比落值区间：
  - **底面**：陆地与海洋统一深色底；**昼端（次日点方向）色 `#1f4468`、夜端色 `#0d1827`，两端均显式 pin**；陆地不单独填色（仅由海岸线勾勒，「陆地是否与海洋异色」裁定为否）。昼夜混合在两端间插值（SPEC-3.2①）——夜端 hue 偏靛、昼端偏青，插值自然完成跨晨昏线的冷色相位移；夜端仍暗于昼端（QA 可断言两端 pin 色值、并断言夜端暗于昼端）。
  - **海岸线**：昼端线色 `#6690b3`（降饱和蓝灰，S≈34%），作**结构线**读、与 news 分类色 `#40a9ff` 拉开饱和度不相混；夜端按 SPEC-3.2① 压暗但保留可辨勾勒。
  - **经纬网格（graticule，程序化生成）**：**默认远景画面不显示经纬网格（D24 默认隐藏）**。网格仅在近景 LOD/调试档启用——**随相机距离的网格显隐规则属视觉第二批（另案），本条不定**；启用时线色 `#1e3a5f`（暗蓝）、密度经线每 30°/纬线每 30°、赤道与极区不额外加密。
  - **夜面**：以海岸线在夜半球的微弱自发光辉光表达（辉光色 `#3a5a72`，低强度；QA 断言其存在与色相，不断言强度精确值）；矢量默认风格无夜纹理/城市点数据源，不含城市级点数据（SPEC-3.3 的「点/辉光」在默认风格下取辉光实现）。
  - **昼夜半球对比契约（默认远景画面对外可验收质量，D24）**：默认矢量风格远景画面须满足——① **主判据**：晨昏线两侧对称代表性采样点的底面 sRGB 亮度比落入 **[1.8, 2.6]**（< 1.8 判为不可辨/平涂、> 2.6 判为过硬如探照灯而失宁静）；② **副判据**：昼半球内部「次日点亮度 : 近晨昏线昼侧亮度」**≥ 1.3 : 1**（防经纬网默认隐藏后昼半球「空」）。**采样几何（视角、采样点角距/同纬度约束）与亮像素排除阈值等量测方法属 testplan/QA 验证方法**（比照 SPEC-3.10「验收基准环境与量测方法由 testplan 场景定义」的先例），spec 只 pin 落值区间。
- **SPEC-3.3** 夜景灯光：夜半球显示夜纹理城市灯光（亮度增益 ≥1.5）；昼半球不显示灯光。
  城市灯光为**卫星/天气风格包专属**（依赖夜纹理）；矢量默认风格以矢量点/辉光表达夜面，不依赖夜纹理。「昼半球不叠加灯光、昼夜为混合而非叠加」的约束跨风格保持。
- **SPEC-3.4** 大气：菲涅尔边缘辉光，主色 `#4a90d9`，从球缘向外衰减；不遮挡标记。
- **SPEC-3.5** 星空：程序化点星 ≥1500 颗，分布于半径 ≥40 球壳，随相机旋转（不随地球自转）。**星最大亮度不得高于昼面海岸线（SPEC-3.2a `#6690b3`）——星空为最底层深度、不得盖过结构层与信息层而成为视觉噪声（保 D3「宁静」、信息永远比结构响，D23 L0）；星亮度分布/上限数值/星色属实现自由度。**
- **SPEC-3.6** 纹理经度对齐：格林尼治（lat 0, lon 0）的球面标记必须落在昼纹理的非洲西侧几内亚湾位置（M1 校准场景验证）。

  - **与大气/夜面自洽**：光柱（含 bloom）在深度上叠于大气之上，延续 SPEC-3.4「大气不遮挡标记」；临边柱段与菲涅尔 halo（`#4a90d9`，SPEC-3.4）重叠处以 screen 混合 + Y≤220 避免脏白；柱立于 SPEC-3.2a 被照亮球面上（非纯黑幕），两半球均须可读（夜面靠亮度对比、昼面靠 hue 对比）。
  - 形态须与 **SPEC-3.8** instancing 约束兼容（≥200 标记不逐事件建 Mesh）；具体实现（billboard/几何/shader、精确衰减曲线形态与系数、高斯 σ、白热核权重系数）属实现自由度，归动效批 dev design-prompt，不入本条。
- **SPEC-3.7b 静态新鲜度编码（D23 L0「多新」）**：事件年龄 `a = now − ts`（`ts` 见 SPEC-6.1，事件时间）经**发光乘子** `F(a) = 0.5 + 0.5·exp(−a/τ)`（`τ = 8h`，**下限 0.5**）调制标记的**辉光热度**——静止画面即可读「多新」，且与 severity（柱高）正交。
  - `F` **只乘辉光**：根部发光基线、白热核权重、大气散射 bloom、根部足印强度（SPEC-3.7a）；**不乘**柱身 pin 基色、柱高、呼吸 alpha（SPEC-3.11）。
  - 观感锚点：a=0（刚发生）→ F=1.00（发烫、白热核满，若 severity 够）；a=24h → F≈0.52（近余烬、辉光弱、无白核，但柱身分类色与柱高仍在）；a ≳ 1 天 → F=0.50 地板。
  - **地板不为 0**：新鲜度**只调辉光不调柱身**，旧事件仍是可辨的分类色柱，**不因变旧而消失**——离屏交给过期离场（SPEC-6.3①），不交给新鲜度。
  - 具体采样/量测方法（`now` 取值、`ts` 缺失兜底见 SPEC-6.1、断言阈值）由 testplan 定义（比照 SPEC-3.10）。
- **SPEC-3.7c 标记 LOD 与密集区聚合**（修 AES-001 M3；**与 SPEC-3.2a 的经纬网格/国界/地名 LOD 为不同对象**——本条管事件标记对象的显隐与聚合，SPEC-3.2a 管底图注记要素，二者互不覆盖、互不推导）：以相机距 `d`（R=1，含初始 3.2，取值区间 SPEC-7.2 [1.8,6]）分三档，保证每档 3 秒读出「哪里·什么类·多新」而不叠罗汉。三档均落在 SPEC-7.2 相机可达区间内。
  - **远** d ≥ 3.0：以根部足印为主——sev1 收为足印点（不出全柱），sev2/sev3 出短柱；同 LOD 下角距 < **5°** 的事件聚为一簇。
  - **中** 2.4 ≤ d < 3.0：全 severity 出全柱、高度阶梯完整可读；角距 < **2.5°** 才聚。
  - **近** 1.8 ≤ d < 2.4（1.8 = SPEC-7.2 变焦下限）：全部独立全柱、体积细节；角距 < **0.8°** 才聚。
  - **代表柱规则**：簇内取**最高 severity** 成员定柱高（高度 = max，平手取**最新**成员），辉光热度取代表成员 age；**hue 不做平均**（取代表成员分类色，不混合多类 hue）。
  - **层级边界**：计数徽章/簇内展开属注记层（L1/label，另案）；L0 每簇只出一根代表柱（守 D23 L0 三变量封顶）。

- **SPEC-3.8** 性能：桌面 Chrome 目标 60fps；标记 ≥200 个时用 instancing/点精灵，不逐事件建 Mesh。
- **SPEC-3.9** 风格与风格包：默认风格 = 轻量矢量（免大纹理，兑现首包预算 SPEC-3.10）。付费风格包 = **天气云图版**（卫星底图 + 定期从免费源更新的气象云图叠加，数据源待调研并写入 ASSETS.md）、**像素**、**艺术**。**定价：¥6 / $1.99 一口价，永久解锁全部付费风格（天气/像素/艺术）**；解锁状态本地持久化（SPEC-8.8）。风格包资源**懒加载**：未解锁/未选用不下载，不计入首包。风格切换入口在设置面板（SPEC-2.4）。
- **SPEC-3.10** 性能预算（与 SPEC-3.8 并列）：中端手机冷启动 → 可交互地球 **≤3s**；缓存事件上屏 **≤1s**（SPEC-3.11）；首包 **≤2MB**（不含可选纹理/风格包）；风格包与卫星/天气纹理一律懒加载，不计入首包。验收基准环境与量测方法由 M5 testplan 场景定义（M2 建基线量测）。
- **SPEC-3.11** 缓存优先启动与呼吸式过渡：冷启动即渲染上次本地缓存事件（球面标记），不空屏等待网络。首次刷新完成后以**呼吸式过渡**收敛：已过期/被替换的旧标记渐隐熄灭、最新事件渐亮，不做整屏重绘闪烁。**呼吸式过渡以「已有在屏状态」为前态、只表达对该前态的增量（旧标记离场、新标记登场，D27）；当无任何前态在屏（全新安装无缓存，或本地缓存为空/读取失败）时，首个非空事件批次直接上屏——标记透明度即置满、不淡入，与 SPEC-3.2①「资源加载完成后直接替换、无淡入过渡」及 D27「呼吸只用于增量、初始/常驻态不施加动画」一致；该首批上屏后，其后所有增量（新增/过期/替换）再按呼吸式过渡收敛。** 缓存从「可重建、不承诺」升格为**启动路径的一部分**（SPEC-8.4）。
- **SPEC-3.11a 减弱动态降级（reduced-motion，P1）**：**稳态无常驻动画**——常驻标记（光柱/辉光/白热核/根部足印）与底图均为静态，稳态截图两帧无差异（无脉冲、无闪烁、无相机驱动微闪）。OS `prefers-reduced-motion` 开启时，**增量呼吸（SPEC-3.11）瞬切（snap）**：登场 alpha 0→1、离场 1→0 瞬时完成、无淡入淡出（含可选高度缓入亦瞬切）；新标记直接以满态出现、过期标记直接消失。**除该一次增量呼吸外，全系统不得存在任何常驻动画**（若引入须一并纳入本条降级）。

## 4. 天文计算（晨昏线）

- **SPEC-4.1** 太阳直下点纬度 = 太阳赤纬 δ（Cooper 公式）：`δ = 23.44° · sin(2π·(284+N)/365)`，N 为年积日。精度要求 ±1°（视觉用途）。
  N 为年积日，按 **UTC 日期**计，1 月 1 日 N=1；闰年 2 月 29 日照常顺次计入（公式分母恒为 365，不因闰年调整）。SPEC-4.2 的 N 取同一定义。
- **SPEC-4.2** 均时差（分钟）：`EoT = 9.87·sin(2B) − 7.53·cos(B) − 1.5·sin(B)`，`B = 2π·(N−81)/364`。
- **SPEC-4.3** 太阳直下点经度 = `−15° · (h_UTC − 12 + EoT/60)`，h_UTC 为 UTC 小数小时；结果归一化到 (−180°, 180°]。
- **SPEC-4.4** 单测锚点（容差 ±1°）：春分 3-20 δ≈0；夏至 6-21 δ≈+23.44°；冬至 12-21 δ≈−23.44°；均时差全年幅度 |EoT| ≤ 17 分钟；EoT≈0 的日期（如 4-15 前后）UTC 正午直下点经度 ≈ 0°。
- **SPEC-4.5** sunDir 向量 = 直下点 (lat, lon) 经 SPEC-6.2 坐标约定转成的单位向量；shader 每帧按当前时刻更新（可降频至 1 次/分钟）。

## 5. 数据源契约（全部免费、无需 key）

统一约束：**SPEC-5.0** 每源独立轮询与限流预算；HTTP 失败指数退避（基础间隔×2^n，上限 30min）；支持 ETag/Last-Modified 的源带条件请求；任何源故障不得影响其他源与渲染。

- **SPEC-5.0a 呈现门槛（surfacing floor，跨源统一约束）**：每源可声明一个**显著性门槛**；仅达到/高于门槛的归一化事件进入**呈现集**——同时决定球面标记（§3）与事件流列表（SPEC-2.2/2.2a）是否出现该事件，**球面与列表共用同一呈现集**（维持 SPEC-2.2「事件流」对球面的从属语义）。低于门槛的事件**既不入球也不入流**。门槛以最省的可用手段实现——源侧提供预筛（feed/查询参数）时优先源侧（见 SPEC-5.1），否则客户端谓词过滤；**实现位置属实现自由度**，对外契约仅为「亚门槛事件不呈现」。**无噪声问题的源不声明门槛（呈现全部事件）**：EONET/GDACS/LL2 现均无门槛。新增或变更任一源的门槛属 spec 变更（§7），须以**实证噪声**为据。呈现门槛只做**准入**（二值 in/out），与密集区**聚合**（D23 L0）、列表行 severity **轻重编码**（SPEC-2.2a）分属不同机制、互不替代。

- **SPEC-5.1 USGS 地震** → category `disaster`
  - `https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/2.5_hour.geojson`（启动时先拉 `2.5_day.geojson` 回填）——采用 USGS **M2.5+ 显著性 feed**，在源侧滤除 M<2.5 仪器级微震，兑现 SPEC-5.0a 呈现门槛（**USGS 门槛 = M≥2.5**）并降低载荷（SPEC-3.10）。
  - 轮询 60s。映射：`id=usgs:{feature.id}`；title=`M{mag} {place}`；lat/lon=geometry；ts=properties.time；url=properties.url。
  - severity：mag<4.5→1，4.5≤mag<6→2，mag≥6→3（2.5 feed 下 severity 1 区间收窄为 [2.5,4.5)，映射规则本身不变）。
- **SPEC-5.2 NASA EONET 自然事件** → category `disaster`
  - `https://eonet.gsfc.nasa.gov/api/v3/events?status=open&days=7`，轮询 300s。
  - 映射：`id=eonet:{event.id}`；坐标取 geometry 数组中**时间最新**的一条——其 `type` 为 `Point` 时取该点 `[lon, lat]`；为 `Polygon`/`MultiPolygon` 时取该 geometry **全部坐标点的经纬度包围盒中心** `((minLon+maxLon)/2, (minLat+maxLat)/2)` 作为 (lat, lon)（`Point` 为其单点退化情形）。该降维为可视化落点近似、不追求面积质心；跨 ±180° 经线的多边形（EONET 极罕见）落点可能偏移，属已知限界，如需可后续另行提案精化。categories[0].title 进 summary；sources[].url 为信源。severity 默认 2。
- **SPEC-5.3 GDACS 灾害/人道** → category `disaster` 或 `humanitarian`
  - `https://www.gdacs.org/gdacsapi/api/events/geteventlist/MAP`，轮询 300s。响应为 GeoJSON FeatureCollection；**同一 `eventid` 出现多条要素**（1 条 `geometry.type='Point'` 中心点 + 若干 Polygon 影响区 / LineString 路径）。归一化按 `eventid` 分组，每组产出一个事件（去重键 `gdacs:{eventid}` 亦合并同 eventid 的多要素）。
  - 坐标：取该 eventid **全部 `geometry.type='Point'` 中心点要素**坐标的经纬度包围盒中心 `((minLon+maxLon)/2,(minLat+maxLat)/2)`（单点退化为该点本身）；Polygon/LineString 要素仅为几何细节，不单独成事件、不参与取坐标。
  - 字段（事件级字段跨同 eventid 各要素一致，取该 eventid 的 Point 中心点要素 `properties`）：`id=gdacs:{eventid}`；title=`name`；summary=`htmldescription`（纯文本，无 HTML 标签）；urls=`[url.report]`；ts=`datemodified`——**GDACS 时间戳为 UTC 且无时区后缀，归一化须显式按 UTC 解析（补 `Z` 或等价），不得依赖 `Date.parse` 的本地时区解释**。
  - severity：`alertlevel` Green/Orange/Red → 1/2/3。
  - category：`eventtype ∈ {DR, FL}` → `humanitarian`，其余（EQ/TC/…）→ `disaster`。
- **SPEC-5.4 GDELT 突发新闻/冲突** → category `news` 或 `conflict`（M3 起点细化查询词与判类规则，此处 pin 接口形态）

- **SPEC-5.5 Launch Library 2 火箭发射** → category `launch`（M2：属 T1 自带坐标源，随首批 provider 接入）
  - `https://ll.thespacedevs.com/2.2.0/launch/upcoming/?limit=10&mode=detailed`，轮询 1800s（免费额 15 req/h，预算 ≤2 req/h；`mode=detailed` 仅增大单响应体，请求数不变仍 2 req/h）。改用 `mode=detailed` 因 `mode=list` 响应不含发射工位坐标。
  - 字段映射：`id=ll2:{results[].id}`；坐标取发射工位 `pad.latitude`/`pad.longitude`（字符串数值，parse 为 number）；title=`name`；summary=`mission.description`；urls 取 `infoURLs[].url` ∪ `vidURLs[].url`，二者皆空时回落自链 `url`（保证 urls≥1）；ts=`net`（T-0，ISO 含 `Z`，`Date.parse` 直接得 UTC）。
  - severity：以 `net` 相对当前时刻的**剩余时间（仅未来方向）**分档——T-1h 内 3，T-24h 内 2，其余 1；**`net` 已过去（发射已发生，`net ≤ now`）时归「其余」档 = 1**，不取绝对值双向对称、不因刚发射而按时间距离判为高档（记录仍在则以最低紧迫度留屏）。
- **SPEC-5.6 OpenSky 航班图层** → category `flight`（**原生端（Capacitor）专属能力，M6**；默认关闭，开启才轮询）

  - `https://opensky-network.org/api/states/all?lamin=…&lomin=…&lamax=…&lomax=…`（当前视口 bbox），轮询 60s，匿名额度内；关图层立即停拉。severity 恒 1。

## 6. GeoEvent 模型与坐标约定

- **SPEC-6.1** 归一化模型（全源统一）：

```ts
interface GeoEvent {
  id: string          // "{source}:{原始id}"，全局唯一，跨轮询去重键
  category: 'disaster' | 'conflict' | 'humanitarian' | 'news' | 'launch' | 'flight'
  severity: 1 | 2 | 3
  title: string
  summary: string     // 信源自带描述；可为空串
  urls: string[]      // 信源链接，详情卡展示，≥1（flight 可为空）
  lat: number         // WGS84 度
  lon: number
  ts: number          // epoch ms（事件时间；无则用抓取时间）
  source: string      // 'usgs' | 'eonet' | 'gdacs' | 'gdelt' | 'll2' | 'opensky'
}
```

- **SPEC-6.2** 球面坐标约定（three.js 右手系，y 上）：北极 (90,·)→+Y；(0,0)→+Z；(0,90°E)→+X。实现：`src/globe/geo.ts` latLonToVector3。
- **SPEC-6.3** 同 id 事件再次出现视为更新（覆盖 ts/severity/summary），不新增标记。过期与保留：①默认过期窗 **48–72h 无更新移除**——「无更新」以事件**最后一次被写入存储（upsert）的墙钟时刻 `lastSeen`** 为过期计时基准，而非事件时间 `ts`：事件只要出现在某轮源响应中被 upsert 即视为「见到」，其 `lastSeen` 刷新、过期计时重置（续期）；连续 48–72h（具体值视本地存储预算定，可配）未再被任何源 upsert 才移除。`lastSeen` 与 `ts` 相互独立——`ts`（事件时间，SPEC-6.1）仅供展示与排序、不参与过期判定，故长寿命事件即便 `ts` 陈旧超窗，只要仍被源持续返回即续期留屏、不被误清。**冷启动**从本地缓存回填的事件沿用其持久化的 `lastSeen`（关机前最后见到时刻），离线时段一并计入「无更新」时长；回填后首轮刷新按 SPEC-3.11 呼吸式过渡收敛（仍被返回者续期、超窗未返回者熄灭），滚动窗口源（如 USGS `all_hour`）的累积窗跨重启不被清空。flight 60s 不变（同以 `lastSeen` 计窗）。②**用户收藏的事件永久本地保存**，不受过期窗影响（M4）。③为未来时间滑块**预留缓存窗口**（滑块 UI 不进早期版本，见 §9）。

## 7. 交互规格

- **SPEC-7.1** 拖拽旋转：水平绕 Y 无限制；垂直视角限制在纬度 ±85°；释放后惯性衰减（阻尼系数 ≈0.95/帧，**以 60fps 为基准**）。
  拖拽与其后的惯性作用于**相机**（绕球心的方位角/仰角），地球本体不因拖拽转动。
- **SPEC-7.2** 缩放：滚轮/双指，相机距离 ∈ [1.8, 6]。
- **SPEC-7.3** 空闲自转：无输入 10s 后缓慢自转（≈0.02°/帧，**以 60fps 为基准**）；任何输入立即停。
  空闲自转作用于**地球本体**（绕地轴，即 SPEC-6.2 的 +Y 轴），相机与星空不动；自转期间晨昏线与地理位置的相对关系保持不变（晨昏线由 SPEC-4.5 的真实时刻驱动，不随自转移动）。
- **SPEC-7.4** 点击标记/列表条目：相机 800ms 缓动飞行到该事件上空 + 弹详情卡（SPEC-2.3）；列表 hover/选中与球面标记高亮双向联动。
- **SPEC-7.5** 时间基准：SPEC-7.1 / 7.3 中以「每帧」表述的常量均以 60fps（帧间隔 1/60 s）为基准；实际帧率不同时，单位**时间**内的衰减比例与自转角速度保持一致，不随设备刷新率变化。

- **产品宪法·永不引入（world-watch constitution，D31 R10）**：以下注意力经济器官**写死为永不引入**，**不适用本节其它条目的「阶段二/另行提案」翻案通道**（本条是负向人格约束，非后置路线，故不因未实现而软化）——
