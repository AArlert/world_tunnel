## 3. 三歧义逐条裁决

### 歧义 1 · 零 chrome 与六开关首屏可见 → **采读法 A（六开关不退位，SPEC-2.4① 不动）**

- **依据**：① product-vision §6 R3「动的是什么」列**仅 SPEC-2.2**，未列 SPEC-2.4①——product-vision 落到可操作层时只指向面板；② REV-018 §1.4 已将 R3 scope 锚定为 SPEC-2.2；③ 极简/外科手术——六开关退位触 SPEC-2.4①/M2-19/FM-10，属独立更大改动、非 R3 必需；④ 重铸文把「零 chrome」精确化为「无常驻事件流面板」+ 显式声明六开关不改，**避免 spec 内部与 SPEC-2.4① 相撞**（正确的 spec 卫生）。
- **裁决**：读法 A。重铸文已 codify，**无需整改**。
- **遗留（surface 给 orch/用户）**：product-vision §4.3 字面「零 chrome」与裁准后仍保留六开关的默认态之间存**已知 gap**。六开关是否也退位（进抽屉/隐藏）是独立产品决策，本批 defer；因 product-vision 字面确写「零 chrome」，建议 orch 向用户明示此 defer（用户反馈为最高优先级输入，CLAUDE.md §0）。见 §6 遗留 2。

### 歧义 2 · 安好态文案在默认收起态的呈现 → **采读法 A（R5 限 SPEC-2.2a 列表空态；收起态额外呈现属 aes/实现自由度，非硬性）**

- **依据**：product-vision §3「它的一天」line 85 把**默认视图的安好**表达为**无光柱的球面**（视觉 absence：「没有光柱，你守望的世界安好」），**非**一句文案；旗舰句「你守望的世界，此刻安好」本体是**列表空态**（SPEC-2.2a），随抽屉展开可见。默认视图安好由 R3 静谧球面（无守望命中→无光柱）承载，二者分工自洽。读法 B 会造「收起态安好呈现载体」新对外行为——越 R5 task scope（=SPEC-2.2a 空态）且 over-build。
- **裁决**：读法 A。**无需整改**（提案 §3 已将「收起态是否额外呈现」标 aes 制定项/实现自由度，与 A 一致）。

### 歧义 3 · 分类全关导致空列表是否套安好态 → **采读法 A（套安好态），但附可信耦合约束 + 遗留**

- **裁决**：本批采读法 A（有守望对象 + 列表呈现集空 → 套安好态，不新增「已隐藏全部类别」区隔特例）。
- **依据（校正提案说理）**：提案的读法-A 理由「叙事不假（你守望之物此刻无呈现）」**说理偏弱**——它把旗舰句的**世界断言**（「你守望的世界安好」）偷换为**呈现断言**（无呈现），二者不等价。**真正支撑 A 的是可信透明性**：因裁准歧义1-A 保留**六开关首屏可见**，用户分类全关时六开关呈 off 态即**可见的收据**（filter 状态透明、非隐藏谎言，比照气压计的可见量程设定）；叠加极简（不新增文案特例），故本批采 A。
- **可信耦合约束（硬）**：歧义3-A **成立以歧义1-A 为前提**。若歧义1 日后翻 B（六开关退位/隐藏），安好态叠加隐藏的 filter 即构成真·失真（product-vision §4.2「不知道就说不知道」/论纲二「气压计不为盖住它的日子谎报无风」），**歧义3 须同步重裁**。orch 若改歧义1，必须回看本条。
- **遗留（可信 watch-item）**：更隐蔽者非「全关」而是**分类部分关恰好隐去了本应呈现的守望命中**（M3-12：分类关闭后即使命中守望也不呈现）→ 列表空 → 安好态谎报「世界安好」而实有命中被藏。本批未强制处置（超歧义3 全关问句、且属边界态），但登记为**可信遗留**：产品宪法级可用性验收（product-vision §8「合上之后更平静还是更焦虑」）若显此为误导，**预定整改路径 = 经 §7 精化安好态触发（排除分类过滤导致的空）或采读法 B 区隔提示**。见 §6 遗留 3。
