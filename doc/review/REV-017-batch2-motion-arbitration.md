## 1. 逐项审查结论

### 审查 1 — reduced-motion（P1）齐备：**通过**

SPEC-3.11a（提案 §2.5）落地两项对外可验收条款：① **稳态无常驻动画**——常驻标记（光柱/辉光/白热核/根部足印）与底图均静态、稳态截图两帧无差异（无脉冲/无闪烁/无相机驱动微闪）；② `prefers-reduced-motion` 下**增量呼吸瞬切（snap）**——登场 alpha 0→1、离场 1→0 瞬时完成、无淡入淡出（含可选高度缓入亦瞬切）。末句「除该一次增量呼吸外全系统不得存在任何常驻动画（若引入须一并纳入本条降级）」把「稳态无动画」升格为对外可验收契约、并预置未来动画的降级挂钩。**REV-013 强制项①（reduced-motion P1 交互条款）兑现**，且与 SPEC-3.2①「初始无淡入」、SPEC-3.11「呼吸只用于增量」自洽。

### 审查 2 — 删「sev3 必须持续脉冲环」的一致性论证：**通过**

提案 §1.2 给出四支论证，非「只删不论」：(1) 该硬句是常驻动画硬命令，与 D27（用户拍板「常驻标记不施加持续动画」）、BUG-031（用户实测否决）正面冲突，按 §7 必须删；(2) 其原服务的 severity 显著性（现行行 95「发光为最强权重杠杆」立法意图 = 破 BUG-022 满屏同权）由**柱高阶梯 + 静态发光基线/白热核**两条静态通道冗余接管，删后不减反增；(3) 脉冲非 severity 语义载体，显著性可靠体积/光/高度（静态）达成，符 D3/D30；(4) 删后全系统唯一残留「动」= 增量呼吸（SPEC-3.11，服务信息非装饰），reduced-motion 下亦瞬切。**REV-013 强制项②（与现硬 spec 的关系论证）兑现**，与 D27「常驻不施动画」一致性成立。改写文本同步删除「脉冲光环幅度随级别递增」半句（现行行 78），二处常驻脉冲语义一并清除，无残留。

### 审查 3 — 色值表零漂移：**通过**

比对提案 §2.1「建议」文本与现行 SPEC-3.7（行 67-95）：**禁改项逐条未动**——① 分类六色色相表（行 69-76）原样；② HSL 乘子 `饱和 S×`（1.00/0.82/0.60）、`明度 L×`（1.00/0.93/0.82）、`透明度`（1.0/1.0/1.0）三列原样；③ category×sev 派生 hex 分级表（行 86-93）原样。提案唯一改动的是**发光列**（脉冲/柔光环/无辉光 → 静态辉光基线 + 白热核）与形态/权重句——发光列不在禁改清单内，且其改动正是删脉冲的落点，非色值漂移。柱身基色仍取现 pin 的 category×sev hex，M3-01（列表行镜像球面 severity 变换，依赖 HSL 分级值）不受影响。**零色值越界。**

## 2. 提案 §6 遗留风险逐项裁决

1. **reduced-motion 编号（3.11a vs 3.12）**：**裁定采 SPEC-3.11a**。该条本质是对唯一动效（增量呼吸，SPEC-3.11）的降级契约，语义内聚于呼吸条，紧随其后合项目子条编号惯例（SPEC-3.2a/3.7a）。末句「若引入任何常驻动画须一并纳入本条降级」已覆盖未来跨全局动画，无须独立成 3.12。非阻断，编号定为 3.11a。
2. **辉光取值 pin 粒度**：**裁定维持提案粒度**。pin 形态尺寸/柱高/足印/tint 色/白热核体积上限/Y≤220/混合模式 + **发光相对档序（sev3>sev2>sev1 硬约束）** + 新鲜度 F；精确衰减曲线函数与系数归 dev DP（承 REV-013 改动 6/7 先例）。特别裁定 **fresh 峰值 Y 210/150/105 维持「绝对值为量测参考、相对档序为硬约束」**（提案 §6-2 自设的取舍分支），不上移为硬 pin 绝对值——绝对亮度随合成/tone-map 实现而异，硬 pin 脆弱；相对档序方为稳定设计意图，与 SPEC-3.10/REV-013 改动 7 同款切分。Y≤220 天花板与白热核体积上限为对外质量锚点（防过曝白/防吞没分类色），准予硬 pin。
3. **τ=8h 与地板 0.5**：**裁定按 aes 值 pin**。二者为 aes 锁定的对外可见新鲜度阈值，须入 SPEC-3.7b（否则 D23 L0「多新」在批次交界静默蒸发）。后续如需按信源差异化衰减速率，属 SPEC-3.7b 后续 spec 变更、走 §7 另案，不在本批范围。
4. **柱高 0.15R 出大气壳的深度实现**：**裁定 spec 层保三点自洽即可**（见审查 7）；深度写入/混色实现归 dev DP。若实现中发现与 SPEC-3.4 难以调和的深度冲突，属实现反馈，回本提案修订或另登缺陷（bugs.md），不在本仲裁范围。
5. **AES-review 判据演进（硬边轮廓判不达标）**：**裁定不在本 spec 提案范围**。aes §8 建议把「硬边轮廓（RingGeometry 式 reticle）判不达标」补入 AES-review「深底上的事件色层次」判据——属 aes/rev 的验收判据（skill）演进，非 spec 契约。登记转 orch 派 aes 处理 `/aes-review` 判据演进，与本 spec 应用解耦。
6. **BUG-031 关单依赖**：**确认**。本提案经本仲裁整改后通过 → orch 应用删脉冲 spec + pin → dev 实现动效批 → qa 重测关单（见 §BUG-031 处理）。SPEC-3.7c 整改点与 BUG-031 核心（删脉冲）正交、不阻塞其关单。

## 3. 整改点（阻断 SPEC-3.7c 应用，其余四落点不受阻）

**整改点 R-1：SPEC-3.7c 近档 `d<1.8` 与 SPEC-7.2 相机距离下限冲突、不可达。**

- **事实**：SPEC-7.2（行 193）pin 相机距离 ∈ **[1.8, 6]**（下限 1.8）；SPEC-3.1（行 48）初始距离 3.2。提案 SPEC-3.7c 三档为 远 `d≥3.0` / 中 `1.8≤d<3.0` / 近 `d<1.8`。**近档 `d<1.8` 在 [1.8,6] 内不可达**——相机最近只能到 d=1.8，此时落中档（`1.8≤d<3.0` 含 1.8）。近档的「独立全柱 + 聚合角距 <0.8°」永不触发。
- **§5.2 后果**：SPEC-3.7c 近档为本提案**新增**子句；因不可达，**无任何 testplan 场景能覆盖**它（qa 无法把相机拉到 d<1.8 验证 0.8° 聚合），该子句自诞生即掉出全部场景、静默蒸发。按 §5.2「会掉出的一律驳回」，此近档条款不得原文应用。
- **整改要求（arch/aes 处理，二选一）**：
  - (a) **aes 确认并上移中/近分界**至相机可达区间内（如中/近分界从 1.8 提高到某 `d0∈(1.8,3.0)`，使近档 `1.8≤d<d0` 可达），arch 转写落 SPEC-3.7c——因分界值是 aes 对外可见取值，须 aes 出值、非 arch/rev 自拟；或
  - (b) 若设计确以当前 SPEC-7.2 下限为准、近档为**未来变焦下限降低后的预留**，则 SPEC-3.7c 明确标注「近档在 SPEC-7.2 现行下限 1.8 下不可达、为预留档」，并**不登记近档必测场景**（避免蒸发式假 🔲）；此时中档承担全部近景体验（其「全 severity 出全柱、高度阶梯完整可读」已覆盖 d=1.8 最近景）。
- 本仲裁**预授权**上述二选一的局部修订方向：arch 据此产出仅限 SPEC-3.7c 近档边界的最小 delta（附 aes 出值或预留标注）后，无须重启完整仲裁，orch 可依本记录 + 该 delta 应用。远/中两档、代表柱规则、聚合角距 5°/2.5°、与 SPEC-3.2a 不同对象的辨析等 SPEC-3.7c 其余内容**准予应用**。

**非阻塞附注 N-1（testplan 补测）**：qa 新登质量场景须显式覆盖 SPEC-3.7a「两半球（昼/夜面）柱均可读」子句（夜面靠亮度对比、昼面靠 hue 对比），防其无场景覆盖而蒸发（§5.2）。

**非阻塞附注 N-2（应用对位）**：提案 §2.1 标现行 severity 句为「line 77」，实际在现行 spec 行 78；提案「建议」引用的现行文本内容准确。orch 应用时按**内容匹配**替换，不按行号。
