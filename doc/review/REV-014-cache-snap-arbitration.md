## 1. 歧义陈述

SPEC-3.11 现行全文：

> **SPEC-3.11** 缓存优先启动与呼吸式过渡：冷启动即渲染上次本地缓存事件（球面标记），不空屏等待网络。首次刷新完成后以**呼吸式过渡**收敛：已过期/被替换的旧标记渐隐熄灭、最新事件渐亮，不做整屏重绘闪烁。缓存从「可重建、不承诺」升格为**启动路径的一部分**（SPEC-8.4）。

条款描述的是「**有缓存**」的正常路径：冷启动即渲染缓存 → 首次刷新完成后以呼吸式过渡收敛（旧渐隐、新渐亮）。未明说的边界：**全新安装无缓存（或本地缓存为空 / 读取失败）时，首个非空事件批次（首轮网络结果）上屏应 snap（透明度即置满）还是淡入（渐亮）**。

现实现（FM-09，`src/globe/markers.ts` 的 `hasPopulated` 标志，行 377 `this.slotAlpha[i] = this.hasPopulated ? 0 : 1`）取「首个非空 `setEvents` 直接置 alpha=1（snap），其后增量才呼吸淡入」。dev 于 FM-09 交付如实上报此边界未落 spec，未自行改判据——处置合规（CLAUDE.md §1，走 §7）。

## 2. 两读法并陈与三维度论证

### 读法 a：首个非空批次直接 snap 上屏（alpha=1，无淡入）
### 读法 b：无前态时首个非空批次也应渐亮（淡入）

三维度逐条论证：

#### 维度一 · spec 文义

- SPEC-3.11 第二句「呼吸式过渡**收敛**：已过期/被替换的**旧标记**渐隐熄灭、最新事件渐亮」——「**收敛**」一词预设一个「从态」与「到态」；「**旧标记**渐隐」在语义上要求存在旧标记（前态在屏）。无前态在屏时，既无「旧标记」可渐隐，也无「从态→到态」的收敛过程可言。故第二句的过渡语义在文义上**以「已有在屏状态」为前提**，不覆盖「从空到首批」的初次建立。
- SPEC-3.11 第一句「冷启动**即**渲染上次本地缓存事件」——「即渲染」表达**立即上屏、不淡入**的建立语义。全新安装无缓存时，第一句的渲染对象为空，真正首次上屏的内容是首轮网络批次；把它作为「初次建立」而 snap，与第一句「即渲染」的建立语义一致；把它作为「收敛」而淡入，则要借用第二句、但第二句语义前提（前态在屏）此时不成立。
- 关键先例 **SPEC-3.2①**（行 47）：底图「资源加载完成后**直接替换，无淡入过渡**」。项目对「初始态建立」已确立统一美学原则——**初次建立即 snap、无淡入；动效只用于状态间过渡**。读法 a 与该既定原则一致；读法 b 会为「标记初次建立」引入一次与球体底图自身「无淡入」相矛盾的淡入。
- **文义维度裁向 a。**

#### 维度二 · 产品意图（D3 / D27）

- **D27（用户 2026-07-21 拍板）**：常驻标记不施加持续动画；「**呼吸动效只用于增量**：①冷启动缓存→首轮刷新完成后，旧点呼吸离场、新点呼吸登场；②运行中仅对过时/新增更新做呼吸登/离场」。D27 把呼吸明确定义为**针对既有在屏态的增量（delta）语义**。全新安装无缓存的首批不是「对前态的增量」，而是**初始态的建立**；按 D27 的框定，初始建立不属呼吸范围。
- D27 进一步**明示背书 FM-09**：「FM-09 过渡已按此语义实现；分类开关启停的过渡观感被用户确认为正确基准」。FM-09 正是含 `hasPopulated` snap 逻辑的实现。用户拍板 D27 时确认的「正确基准」即建立在「增量呼吸、初始/常驻不施动画」之上——与读法 a 同向。
- **D3（宁静）**：全新安装时用户面对空球，首批 50~200 标记若同步整屏淡入，观感更接近「加载揭幕的花活/loading reveal」，与「界面本身必须令人平静」相悖；snap 建立则安静、无仪式感。D3 亦裁向 a。
  - 补注（边界声明）：首批 snap 的「pop 是否突兀」属**观感/审美**判断，归 aes 而非 rev。本仲裁只裁「呼吸过渡的 spec 契约是否覆盖无缓存首批」这一**契约问题**（答：不覆盖，snap）。若日后 aes 认为需要一次克制的初次柔和揭幕，属**另案新提案**，不在本仲裁范围，且须与 SPEC-3.2① 的「初始无淡入」原则一并权衡。
- **产品意图维度裁向 a。**

#### 维度三 · SPEC-3.10 时长约束

- SPEC-3.10「**缓存事件**上屏 ≤1s（SPEC-3.11）」——该 ≤1s 硬约束的对象是**缓存事件**。全新安装无缓存路径下不存在缓存事件，该子句不直接绑定无缓存首批，故不能单以 ≤1s 反推首批必须 snap。
- 但方向一致：≤1s 对缓存路径要求「即上屏」，本身就把「缓存首批 = 立即建立、不做淡入延迟」钉为契约（`FADE_DURATION_MS`=500ms 的淡入会软化/推迟「上屏」判定）。无缓存首批作为「同类的初次建立」，按一致性取 snap，与 ≤1s 对缓存路径的即时建立取向同构；反之读法 b 会使初次建立独一处引入 500ms 淡入延迟，与全条「即渲染 / ≤1s」基调不谐。
- **时长约束维度：不直接绑定，但方向一致，弱裁向 a。**

### 裁决：采读法 a

三维度（文义强裁 a、产品意图强裁 a、时长约束弱裁 a），**一致裁定读法 a**：

> **全新安装无缓存（或本地缓存为空 / 读取失败）时，首个非空事件批次直接 snap 上屏（标记透明度即置满、不淡入）。呼吸式过渡以「已有在屏状态」为前态、只表达对该前态的增量（旧标记离场、新标记登场）；该首批建立后，其后所有增量再按呼吸式过渡收敛。**

现实现（FM-09 `hasPopulated` snap）与本裁决一致——本裁决为**消歧澄清**（把已实现、且经 D27 用户背书的行为落 spec 文字），非引入新行为，故属 rev 对「spec 歧义条目」的仲裁范围（CLAUDE.md §3/§7），由 orch 应用 + pin。

## 4. testplan 影响裁定

现状核对（`doc/testplan.md`）：
- **M2-20**（缓存优先启动单测，✅，`make test TEST=cache-first-start`）：断言「网络首轮未完成期间 store 已现缓存事件」+「无缓存/缓存读取失败启动不阻塞、不抛未捕获异常」。焦点为**数据层启动顺序与鲁棒性**，不涉标记 alpha 的 snap/淡入。
- **M2-21**（呼吸式过渡 e2e，✅，`make e2e TEST=marker-breathing-transition`）：以**一组缓存事件上屏后**首轮刷新返回差异集，断言旧标记连续渐隐、新标记连续渐亮、无整屏闪烁。焦点为**有缓存的收敛过程**，其三项断言均围绕刷新收敛，**未断言初次建立批次是 snap（无淡入）**。

裁定：

1. **M2-20 维持现范围与 ✅**——其断言（启动顺序、无缓存不阻塞）在读法 a 下全部仍成立，无需改动、无需降级（rev 亦不得自行降级他人 ✅，CLAUDE.md §5.2）。
2. **M2-21 维持现范围与 ✅**——其三项断言针对「有缓存收敛」，与本裁决不矛盾、仍有效；不要求扩写。
3. **要求 orch 派 qa 新登记一条场景**，承接本次新 pin 的对外可见行为「首个非空批次直接 snap 上屏、不淡入」——**理由：防子句静默蒸发**。本裁决为 SPEC-3.11 新增了一条对外可见、可机械验证的行为契约（无前态首批 snap）；现有 M2-20/M2-21 无一断言之，若不补场景，该子句将「掉出全部场景之外」，`make next` 永不再提示、被后续改动（如误把首批也做成淡入）静默破坏而无人察觉（BUG-014 / REV-002 G-1「静默蒸发」同构模式；亦见 BUG-029 先例）。该行为**能写出确定性复现**（标记层 alpha 初值单测），非「写不出复现命令」类，故按 CLAUDE.md §5.3.4 边界须走机械测试证据、不得口头认领。
   - 建议形态（**建议级，具体登记由 qa 判定**）：标记层（`markers.ts`）单测——
     - 断言 A：新建标记层（`hasPopulated=false`）首个非空 `setEvents` 后，新槽 alpha 初值**即为满值（1），非从 0 起坡**（snap，引新版 SPEC-3.11 消歧句 + SPEC-3.2① 初始无淡入）；
     - 断言 B（对照）：`hasPopulated=true` 后新增标记的 alpha 初值**从 0 起、经 tick 坡向 1**（增量呼吸淡入，引 SPEC-3.11 呼吸句 + D27 增量语义）；
     - 期望值均须从 spec 文义推导、标注 SPEC 条目号，**禁止照抄实现**（CLAUDE.md §7；断言 A 的「首批 snap」判据独立源自本裁决文字，非源自 `hasPopulated` 变量存在这一实现事实）。
   - 为何另登新行、不并入 M2-20/M2-21：M2-20 焦点为 store 启动顺序（数据层），M2-21 焦点为有缓存 e2e 收敛；「首批 snap 对照增量淡入」是**标记层 alpha 初值**的独立契约，另登一行保留对新 SPEC-3.11 子句的可追溯认领，避免过载既有两行的焦点。
4. **不阻塞既有 ✅**：本补场景为覆盖补齐、承接新 pin 子句，属整改项而非对 M2-20/M2-21 的返工；不影响其现状态与 M2 既有绿。
