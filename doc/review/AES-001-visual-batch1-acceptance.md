## 2. 做对了什么(先肯定)

1. **黑名单#1 Dark Acid Default 被真正破除**——不是口号。底盘有明度阶梯(空间→夜半球→晨昏→昼半球，`far-day-empty` 可见柔和梯度)，标记按 severity 三通道分层。构造密集 sev1 簇(`far-day-events` 爱琴海 14 枚)与真实环太平洋密集震群(`real-far-pacific-B` 阿留申/汤加)**都**读作「一片安静的哑红点 + 偶尔一颗带环的强震」，而非现状那种满屏同权亮红。这是本批最实的成果。
2. **半球明暗对比达标且柔和**(D24)——`far-terminator-empty`/`mid-terminator-events` 可见昼侧偏亮、夜侧沉暗的整体差；机械场景 M3-02 已绿(比值落 [1.8,2.6])。柔和而非探照灯，符 D24「半球级对比才是目标，晨昏锐利非关键」。
3. **夜面辉光 `#3a5a72` 观感修复**——`mid-night-glow-empty` 显示夜侧海岸线为可辨但克制的蓝灰勾廓，非洲/南美陆廓读得出且不喧宾。**历史关注点(v0.2.5「偏淡待过目」)本批已正向解决**，辉光现在「提示得到、不抢戏」，取值恰当。
4. **海岸线降饱和 `#6690b3` 读作结构线**——与 news 分类蓝 `#40a9ff` 拉开饱和度，`far-day-events` 中海岸线与蓝色新闻标记无混淆，结构层/信息层分得干净(SPEC-3.2a/3.5「信息永远比结构响」兑现)。
5. **列表行 severity 三档明度单调、全档过 WCAG**——`panel-events-crop` 中 sev3 标题(如「Drought in Madagascar」「M5 Mid-Indian Ridge」)明显跳出，sev1(「Drought in Kazakhstan/Brazil」)沉下。实测对比：sev3 15.6–17.7:1 / sev2 10.8–12.2:1 / sev1 5.7–6.4:1，全过 AA 正文 4.5:1。填 SPEC-2.2a D25 留白成立。
6. **面板图底关系正确**——面板底不亮于夜半球底、与球同蓝族、发丝边非重阴影、不平摊盖球面信息(`panel-events`/`real-empty-far-day`)，D3「球为本体」保住。
7. **大气菲涅尔辉光成为凝聚全局的克制 signature**——单色蓝 halo(SPEC-3.4)给出 cobe 式「被点亮的星球」质感，无霓虹、不 timid，是全画面辨识度最高的一笔。
8. **空态自洽**——`real-empty-far-day` 无事件时底盘独立成立、面板「暂无事件」沉静，不空洞、不焦虑。

## 4. 分级问题清单(只提问题、不开药方)

### Medium(放行·登记，供后续批次)

- **M1 · signature 主次：晨昏梯度弱于大气 halo，「the living terminator」未成最记得住的一笔。**
  制定方案宣称的唯一 signature 是昼青→夜靛的晨昏冷色位移；但 `far-day-empty` vs `far-night-empty` vs `far-terminator-empty` 三图对比，最抢眼、最辨识的其实是大气 halo，晨昏梯度柔和到接近一眼掠过。半球对比本身达标(M3-02 绿、[1.8,2.6])且 D24 只要「柔和可辨」——**故不挡门**；但作为「让人记住的一件事」，terminator 目前更像克制的支撑梯度而非 signature。
  判据：品味原则「一个 signature 其余克制」、D24。→ 建议第二批 aes 裁定是否强化冷色位移以坐实 signature，或明确把 signature 归给大气 halo。

- **M2 · 标记 severity 环读作「雷达标线/reticle」，非「柔光/bloom」，近景偏重。**
  `near-day-events`/`mid-terminator-events` 中 sev2/sev3 的环是硬边圆环轮廓(RingGeometry)，近景放大后成较粗的「靶环」，与制定方案「bloom 柔光质感」的意图有出入。**severity 分层功能本身成立**(sev3 有环最响、sev1 无环最静)，问题在环的**形态质感**。
  判据：3D 判据「深底上的事件色层次」(层次达标)、品味原则「辉光质感」。→ 标记形态归 **D27 动效批**(光柱候选)，此条为形态质感佐证，请动效批一并纳入，不另立。

- **M3 · 真实密集震群叠罗汉、无聚合。**
  `real-far-pacific-B`(阿留申/加州)、`real-mid-asia`(汤加/克马德克)密集区标记与环相互重叠成一小片红簇，未做聚合。severity 层次在簇内仍成立、未糊死到不可读，**故非 High**；但指向 D23 L0「密集区须聚合、不叠罗汉」。
  判据：D23 L0、3D 判据「密度随相机分层(LOD)」。→ LOD/聚合明列**视觉第二批**(design-prompt §4.2「密度分层待第二批」)，此条确认其真实性与优先级，不作新问题登记。

- **M4 · L0「多新」变量在静止态未持续编码。**
  D23 L0 要求球面 3 秒读出「哪里·什么类·多新」。当前球面稳态只编码位置(✓)与分类色(✓)，**新鲜度**靠呼吸式登/离场(FM-09/D27)表达、静止稳态下不可读。本批范围(配色/severity/昼夜)未含新鲜度编码、亦未声称交付它，故**不作门禁失败**。
  判据：D23 L0、验收 skill §3 blocker 项(此处按范围豁免，非命中)。→ 归 **D27 动效批 / FM-09** 呼吸机制；请确认该 L0 变量有批次认领，勿在批次交界处静默蒸发。

### Nit(放行·登记)

- **Nit-1 · 相对时间文本对比偏薄。** 实现用 `rgba(232,232,236,0.4)`，实测约 3.28–3.33:1，过注记级 3:1 但余量薄；且与制定方案建议的 `#6d7a8a`(≈4.2:1)有出入。判据：无障碍 ≥3:1。属 SPEC-2.2a UI 自由度，无需改 spec；dev 可考虑抬一档取余量。
- **Nit-2 · 面板用 `backdrop-filter: blur(8px)` 毛玻璃，与制定方案「实底、无模糊玻璃」有出入。** SPEC-2.2 明列「实底或玻璃」属 UI 自由度、且当前玻璃克制无霓虹(**未命中黑名单#3**)，故仅记录。判据：黑名单#3(未命中)、design-prompt §6。请 orch/arch 知悉方案与实现的这一处不一致，择一定调以免后续漂移。
- **Nit-3 · 大气 halo 夜侧球缘同样明亮。** SPEC-3.4 单色菲涅尔已 pin，制定方案的「夜侧大气压暗(可选)」未采纳(方案本就标为可整体砍)。物理上夜侧亮 halo 略反直觉，但观感讨喜、临边轮廓任何角度都勾得出，仅记录、不建议动。
