## 2. 重点复核项（逐项独立判断）

### 2.1 复核 1 — M1-08「星空随相机旋转」的判据强度：**成立，qa 的自我质疑不成立**

qa 的疑点是：大幅拖拽后画面必然变化，故「星空采样区像素发生变化」缺乏证伪能力。**本审查员不采纳该自评**，理由是它忽略了采样区的几何位置与第二个用例构成的对照关系。

**(a) 采样区内不存在星空以外的可变内容 —— 几何论证（独立复算）**

`e2e/starfield.spec.ts:23-28` 的采样区是画布左侧 20% 宽的竖条。`playwright.config.ts` 未设 `use.viewport`，取 Playwright 默认 1280×720；`src/index.css:19` 顶栏占 48px，故 canvas 为 1280×672，aspect ≈ 1.905。以 SPEC-3.1（fov 45°、距球心 3.2、球半径 1）计：

- 水平半视场 tan = tan(22.5°)×1.905 ≈ 0.789
- 地球轮廓视线半张角 asin(1/3.2) = 18.21°，tan ≈ 0.329 → 占水平半宽 41.7%，即横跨画布 29.2%~70.8%
- 大气壳半径 1.15（`src/globe/atmosphere.ts:5`）：asin(1.15/3.2) = 21.05°，tan ≈ 0.385 → 占 48.8%，即横跨 25.6%~74.4%

采样区右边界在 20%，距大气壳最外缘尚有 5.6% 画布宽（约 72px）余量。**该竖条内除星点外无任何被渲染物**。归档截图 `M1-08-starfield.png` 目视复核确认该区域为纯背景 + 星点，与计算一致。

**(b) 相机绕球心旋转不移动地球/大气在屏幕上的位置**

`GlobeScene.applyCamera()` 每帧 `camera.lookAt(0,0,0)` 且 distance 不变，故地球与大气壳的投影轮廓恒为屏幕中心的同心圆、半径不变；大气辉光绕视轴旋转对称。因此拖拽只改变地球表面纹理内容，不会把任何地球侧内容送进左侧竖条。

**(c) 两个用例构成对照组／处理组，互为前提证明**

- 用例 2（对照组）：地球本体持续自转 3 秒，竖条像素 `Buffer.compare(...) === 0` 逐字节相等。这实测证明了该竖条内**不含随地球运动的内容，且渲染在无相机运动时是逐字节确定的**——(a)(b) 的几何论证由此获得实测背书。
- 用例 1（处理组）：仅相机转动，竖条像素改变。在用例 2 已确立的前提下，该变化只可能来自随相机运动的星空。

**可被证伪的错误实现（判据有区分力的证明）**：星空若挂在相机之下（随相机运动、屏幕上静止），用例 1 变红；星空若挂在地球组之下，用例 2 变红；星空若未渲染／竖条全黑，用例 1 变红（黑与黑逐字节相等）。三类主要错误实现均被拦截。

**结论：判据具备证伪能力，不影响 M1 签核，无需整改。**

**附脆弱性提示（非阻塞，登记为 §5 R-5）**：上述安全余量依赖 Playwright 默认视口 1280×720。若日后有人在 `playwright.config.ts` 设置更接近正方形的视口（如 800×800，aspect 1.0 时大气壳外缘将推进到画布 12.2% 处），竖条会吃进大气辉光，用例 1 会因辉光边缘的抗锯齿抖动而**在星空实现已损坏时仍然变绿**——判据静默失效且无人察觉。建议在该 spec 文件头注补一句「本采样区的安全性依赖 config 中的视口宽高比，改视口须重算」，或在用例内加一条前置自检断言视口宽高比下限。属 qa 侧改动，由 orch 派单。

### 2.3 复核 3 — M1-14 的证伪能力与变异验证的可采信性：**可采信**

**(a) 变异实验本身的可信度**：qa 报称把 `mix` 改为叠加式后昼侧 delta 由 0.00 变为 15~24（容差 3）。本审查员注意到 `src/globe/shaders/earth.ts` 的 mtime 为 2026-07-19 21:58:52，**远早于** M1-14 的全部工作时段（2026-07-20 02:00~02:50），且该文件为未跟踪文件（无法用 `git checkout` 无痕还原）。即：**qa 未修改 src/，变异是通过运行时改写 `material.fragmentShader` + `needsUpdate` 触发重编译完成的**，与其「运行时 shader 变异验证」的自述一致，也未越 `.claude/agents/qa.md` 的「不改 src/」边界。此点予以确认并记录。

**(b) 更强的一点：该证伪能力不依赖那次实验，可解析地证明。** 这是本项可采信的根本理由——实验不可复现（无归档 log），但结论可由已归档工件独立推导：

- 现实现 `gl_FragColor = vec4(mix(night * uNightGain, day, k), 1.0)`，其中 `k = smoothstep(-uTwilight, uTwilight, t)`，`uTwilight = 0.1`（`src/globe/earth.ts:6`，即 SPEC-3.2 的过渡带半宽）。测试取样点 t = 0.3 与 t = 1.0，两者均 ≥ 0.1 ⇒ `k ≡ 1` ⇒ 结果恒为 `day`，**对 `uNightGain` 的偏导数解析地等于 0**，与容差 3 之间有无限裕度。
- 叠加式实现 `day + night * uNightGain` 的昼侧偏导数为 `night_raw`，delta = `night_raw × (6.0 − 1.5) = night_raw × 4.5`。要落进容差 3 需 `night_raw < 0.67`（0~255 标度），而用例的取样经度是在候选中**按「夜侧增益响应最大」主动挑选**的（`e2e/day-side-no-lights.spec.ts:127-136`），即刻意选对错误实现最不利的观测窗口。qa 报称的 delta 15~24 反推 `night_raw ≈ 3.3~5.3`，与该窗口选择法一致，数量级自洽。

即：**证伪门槛的满足是可验算的，变异实验只是佐证而非唯一依据**。REV-003 §2.2 第 1 条硬要求（「断言必须在实现被改为昼夜叠加时失败」并在交付汇报中说明）得到满足，且说明被固化进 `e2e/day-side-no-lights.spec.ts:36-43` 的文件头注，不会随交付汇报散佚。

**(c) REV-003 §2.2 的两条明令禁止（断言 shader 源码字符串、移植 GLSL 公式再断言）**：逐行核对该 spec 文件与 `e2e/globeDebug.ts`，仅对已知 uniform 名 `uSunDir`/`uNightGain`/`markerRoot.rotation.y` 读写，判定全部基于真实渲染回读像素，**未触犯任一禁令**。

**(d) REV-003 §2.2 第 4 条（取样点不可写死）**：第二个用例全程不注入 sunDir，由当前 `uSunDir` 反算取样经度，并以 SPEC-4.1 的 |δ| ≤ 23.44° 推出赤道上恒存在 t ≥ cos(23.44°) ≈ 0.917 的深昼点——推导正确，闭合了「可见半球全为夜侧」的偶发红隐患。**处理正确，予以肯定。**

**(e) 仪器有效性对照组**：夜侧必须随增益变化（`nightDelta > 12`）的设计，排除了「两侧都不变 = 采样失灵」的伪绿，是本用例最关键的一笔。**设计正确。**

### 2.4 复核 4/5 — BUG-010 与 BUG-007 是否阻塞签核

3. 已连续 7 次全量复跑未复现，且被 8 worker 并发的全量回归覆盖（`regress_summary.txt` e2e PASS）。

**但须在 M2 内闭合**，理由与 bugs.md 登记的风险陈述一致（偶发红侵蚀证据链可信度）。本记录附**两条可检验的根因假设**供 orch 派单时使用（供排查，非结论）：

- **假设 H1（空闲阈值余量不足）**：`e2e/idle-spin.spec.ts:46-53` 在 6s 处断言「尚未自转」，而 SPEC-7.3 的阈值是 10s，余量仅 4s；这段余量要吸收 `page.goto`、`waitForGlobeDebug`（内含 150ms 等待）与两次 `page.evaluate` 往返。8 worker 并发下滑掉 4s 完全可能，一旦滑过阈值 `earthRotY` 即变，`toBeCloseTo(..., 9)` 立刻红。此假设与 bugs.md 记的「对自转起始时刻的近似比较」在语义上吻合度最高。

**定性与处置**：**不构成「未挣得的 ✅」，不要求回退状态位**。理由：涉及的四条断言均真实存在、真实通过，且被条件 ② 的全量回归（`regress_summary.txt` unit PASS）覆盖，无任何 spec 子句因此掉出全部场景之外——与 REV-003 §1 所处理的「判据零覆盖」有性质区别。但**复跑列不忠实**（照 `make e2e TEST=atmosphere-glow` 复跑无法复现材质代理断言），属证据链粒度缺陷，须登记留痕（§5.3 禁止只在对话里口头传递）。

**期望**：`make evidence SCEN=` 支持归档多个 log 源（例如允许 `--test` 与 `--e2e-test` 并存，或 `--log` 可重复），使跨层场景的证据与复跑命令完整；或明确禁止登记判据跨层的场景、要求 qa 拆行。二选一由 arch/dev 定，rev 不指定实现。**修复后须对 M1-07、M1-08 重跑 evidence 补齐**，可在 M2 首个卡内闭合。
