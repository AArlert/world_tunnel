## 1. 交付门禁逐项结论

### 1.1 门禁 ① 行为泄漏禁区：**通过**（本卡核心门禁项）

独立枚举 DP 每一处对外可见行为，判其是否指回已 pin SPEC 或明确的提案项：

| DP 位置 | 对外可见项 | 归属 | 判定 |

| §2.1 矢量海岸线/网格/免大纹理、卫星退出默认 | 风格构成 | SPEC-3.2②③/3.10 已 pin「存在」 | 指回，无泄漏 |
| §2.1 夜面点/辉光**具体形态·颜色·数据来源**、海岸线线色/底色/网格线色与密度 | 可见取值 | **DP 明确列 P-1、明令 DEV 不得自定，可用明显占位但不得作交付外观** | **正确隔离，无泄漏** |
| §2.2 六 category 色逐字照 SPEC-3.7 | 分类色 | SPEC-3.7 已 pin | 指回，无泄漏 |
| §2.2 severity 尺寸/脉冲递增、sev3 持续脉冲 | 分级 | SPEC-3.7 pin「递增+存在」，具体像素/频率为实现自由度（类比 REV-002 D-1） | 指回，无泄漏 |

### 2.4 SPEC-7.4「双向」读法：**裁准「M2 全双向」（含 marker→list hover 拾取）**

裁决 arch 取"M2 全双向"读法，驳回"拆半（marker→list 延后 M3）"读法。理由链：

1. **spec 明文**：SPEC-7.4 末句"列表 hover/选中与球面标记高亮**双向联动**"是单一短语，"双向"是明文定语，两方向同属一句。
2. **FM 锚点**：feature-matrix FM-07 交付物含"SPEC-7.4 列表↔标记联动分片"——↔ 即双向；FM-14 只领"点击飞行+详情卡分片"。REV-005 K-1 的拆片边界是**按"hover/选中高亮" vs "点击飞行+详情卡"划分**，整个高亮联动（含两方向）落在 M2/FM-07 一侧，点击-动作落 M3/FM-14 一侧。marker→list hover 高亮属高亮联动，天然在 M2 侧。

4. **拾取范围有界、无泄漏**：marker→list 的 raycast 严格限于"求 hovered 事件 id → 驱动高亮"，DP §2.4/§3.5 明令不做点击动作/不飞行/不弹详情卡。这不是行为泄漏（marker hover→list 高亮是 SPEC-7.4 明文），也不是 M3 越界（M3 是点击-飞行-详情卡）。
