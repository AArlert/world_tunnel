## 1. K-1~K-4 逐项消除核验

### 1.1 K-1（SPEC-7.4 里程碑归属）：已消除

独立回验原始事实：
- 现行 `doc/spec.md:115` SPEC-7.4 = 「点击标记/列表条目：相机 800ms 缓动飞行…+ 弹详情卡；列表 hover/选中与球面标记高亮双向联动」——单一条文含两类对外可见行为。
- `doc/design-prompt/M1-globe.md:13`「点击飞行与详情卡（SPEC-7.4）」列入**不做**；`:227`「SPEC-7.4 的相机飞行是 M2/M3 的事…不要预留 flyTo 空壳」。M1 testplan 无任何点击飞行/详情卡/列表联动行。故 SPEC-7.4 = 未交付，v1 标 done 属静默蒸发。

修订核验：
- §2.22 明示「正文不改一字」，仅修订里程碑归属，纠正 v1 §3.2「SPEC-7.1~7.5 done(FM-04)」误判。
- §3.2（line 323）已改为「SPEC-7.1/7.2/7.3/7.5 done(FM-04)；SPEC-7.4 未交付（K-1）→ 列表↔标记联动 FM-07(M2) + 点击飞行 800ms + 详情卡 FM-14(M3)」。误标 done 已根除。
- FM 锚点补齐：FM-07（line 279）含「7.4(列表↔标记联动·K-1)」；FM-14（line 286）含「7.4(点击飞行 800ms + 详情卡·K-1)」。
- §4 覆盖点补齐：M2 列「列表 hover/选中 ↔ 球面标记高亮双向联动（K-1）」；M3 列「点击标记/列表条目 → 相机 800ms 飞行 + 弹详情卡（K-1）」。
- **拆片完整性核验**：SPEC-7.4 正文两类行为（飞行+详情卡 / 列表联动）分别落 FM-14 / FM-07，两端合并全覆盖，无子行为遗漏；两 FM 行各建真实 testplan 行后，条文不再有「呈 done 而永不再被提示」路径。**消除成立。**
