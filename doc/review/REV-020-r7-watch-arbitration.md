## 3. 歧义裁决 · 命中强调读法（仲裁必答项 3）

**裁定：隔离式命中呈现入 M3 / inline 命中强调后置 M4（采提案 §7.1 裁量方向）。**

依 REV-018 §1.1④ 原文与极简纪律，逐条理由：

1. **REV-018 §1.1④ 已将详细读法「亦正确交 R7 §7」**——原文只给形状「『仅命中最小模式 + 命中强调』→M3、『完整过滤模式』→M4」，并未锁死「命中强调」= inline 抑或隔离。本卡（R7 §7）有权细化「M3 命中强调」的落地读法，不构成对 REV-018 的推翻。
2. **隔离式确实兑现 SPEC-8.2 现行子句「命中事件在列表与球面均有视觉强调」**：在「仅守望命中」模式下，命中事件成为列表与球面的**唯一呈现内容**（非命中隐藏）——以排除达成最大化强调，两处（列表+球面）均在。子句得到 honor，非蒸发。
3. **字典税纪律（D23，product-vision §4.3 + §7「不动的」）**：inline 命中强调 = 在「全部」模式下额外标示命中/非命中，是 L0 三变量（哪里/什么类/多新）之外**新增第四视觉轴**，须付字典税（aes 方案定命中态光柱/列表编码取值 → arch 落 SPEC-8.2，类比 SPEC-3.7 光柱先例）。M3 阶段擅立此轴违字典税；后置 M4 恰是付税之地。
4. **极简纪律（CLAUDE.md §1.2）**：守望闭环（引导采集→本地存→守望匹配→命中→仅命中过滤+隔离呈现）在隔离式下已完整闭合；inline 是锦上添花、非闭环必需。M3 复用与分类过滤（SPEC-2.4①）同类的显隐机制、零新增视觉轴，是最小落地。
5. **§5.2 无蒸发**：inline 命中强调完整保留为 M4 SPEC-8.2 + FM-16 承接行，未掉出全部场景（§2.2）。

**裁决效力**：FM-28〔守望最小闭环〕**无 aes visual-pin 前置依赖**，R7 spec 落定后即可驱动 testplan/DEV（与提案 §6.3 F-4 注一致）。inline 命中强调的 aes 付税义务转入 M4/FM-16（届时另开 aes 方案 + arch 落 SPEC-8.2）。
