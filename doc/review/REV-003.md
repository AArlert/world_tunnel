## 2. 争议二：不适用豁免，拆分场景 + e2e 像素判定

### 2.1 驳回「结构性不可验、豁免并 M2 回补」

**qa 的处理正确**（未靠解析 shader 源码字符串假装覆盖，如实保持 🔲）——解析源码字符串断言 `mix(` 是同义反复，只证明「代码写成了它写成的样子」，属 qa.md 输入边界明令禁止的「把实现行为当期望值」。此点予以确认。

但**援引 REV-002 对 SPEC-3.4「不遮挡标记」的豁免先例不成立**，两者性质不同，此处划清界线：

| | SPEC-3.4「不遮挡标记」 | SPEC-3.3「昼半球不叠加灯光」 |
| --- | --- | --- |
| 被断言对象 | **标记层，M1 不存在** | **昼半球，M1 就在屏幕上** |
| 不可验的成因 | 断言对象缺席——任何测试技术都无法验证一个不存在的东西 | 单一工具（vitest）无 WebGL 上下文 |
| 项目是否具备验证能力 | 否，须等 M2 标记层 | **是**，e2e 层为真实 Chromium + WebGL，BUG-001 已解决浏览器可达性，`e2e/smoke.spec.ts` 已跑通 |
| 定性 | 结构性不可验 | **工具选择问题** |

**「vitest 测不了」不等于「M1 不可验」。** 项目的测试体系是单测 + e2e 两层（CLAUDE.md §2、§4），判据落在哪一层是选择而非约束。以工具局限换取 spec 条目的豁免，会开出一条「换个测不了的工具就能豁免」的通用后门，必须在此堵死。

**加重情节**：SPEC-3.3 全文两句——「夜半球显示夜纹理城市灯光（亮度增益 ≥1.5）；昼半球不显示灯光」。第二句正是本判据。若豁免，SPEC-3.3 在 M1（地球仪里程碑，该条的**主场**）只有一半被验证，而 M2 是事件数据层，届时无人有动机回头补一条地球着色断言——REV-002 R-2 已把 SPEC-3.4 记为长期未验证风险，不宜再叠加第二笔。

### 2.2 裁决：拆分场景

采用「拆分」路径（三选项中的第二项），并指定被拆出判据的验证层为 e2e。

**M1-06 现行行文（第 17 行）拆为两行：**

- **M1-06（保留、可立即 ✅）** 建议措辞：

  > 夜景灯光增益下限（单测）：shader uniform `uNightGain` 默认值 ≥1.5（SPEC-3.3「亮度增益 ≥1.5」的机械化断言）。昼半球不叠加灯光的判据见 M1-14。

  说明：删去「另附截图作视觉辅助证据」。REV-002 T-3 当初把截图定为**辅证**，辅证不构成 ✅ 的必要条件；既然主判据已单测覆盖且拆分后本行只剩该主判据，保留一个不影响判定的可选项只会重演 M1-07 的「行文承诺了证据里没有的东西」。**注意本处的删词与裁决 ② 所禁止的删词性质不同**：此处被删的是辅助证据形式，SPEC-3.3 无任何子句因此失去覆盖（第二句完整移入 M1-14）；M1-07 处被要求删的则是 spec 子句本身的唯一覆盖。判别标准即：**改后是否有 spec 子句掉出全部场景之外**。

- **M1-14（新登记，🔲）** 建议措辞：

  > 昼半球不叠加夜景灯光（e2e，真实 WebGL 渲染）：在昼半球内部（t = dot(N, sunDir) 明显大于 SPEC-3.2 过渡带上界 +0.1）取样点的最终颜色**不随 `uNightGain` 变化**，而夜半球取样点随之变化——证明灯光项在纯昼端点的混合权重为 0、昼夜为混合而非叠加（SPEC-3.3 第二句 + SPEC-3.2 混合定义）。

**为何用「对 uNightGain 的不变性」而非「与昼纹理比色」**：后者需要黄金参考图或纹理采样真值，脆弱且引入外部依赖；前者只需同一场景两次取样对比，无参考基准，且**恰好能证伪目标错误**——若实现改成 `day + night*gain` 叠加，昼侧像素必随 gain 变化，断言立刻红。这满足 CLAUDE.md §1.4「把任务转成可验证判据」，也是本记录对「有效判据」的最低门槛。

**给 qa 的边界（避免二次解读）：**

1. **最低证伪门槛（硬要求）**：断言必须在「实现被改为昼夜叠加」时失败。交付汇报中须说明该断言如何满足此条。做不到则不算覆盖，如实保持 🔲 并回报。
2. **明令禁止**：(i) 断言 shader 源码字符串含 `mix`/权重表达式；(ii) 把 GLSL 混合公式移植成 TS 函数再对该移植断言。两者皆为同义反复。

4. **必须处理的坑：uSunDir 由真实时刻驱动**（SPEC-4.5），直下点经度全天扫过所有经度，而相机固定看向 (0,0)（SPEC-3.1）——**存在整个可见半球全为夜侧的时刻**，取样点不可写死，须由当前 `uSunDir` 反算，或先转 `earthGroup.rotation.y` 把昼区转进视野。写死取样点必出偶发红。
